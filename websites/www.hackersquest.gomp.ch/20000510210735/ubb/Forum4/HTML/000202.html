I've got a perl script that I've written which uses GD.pm to create png files of the zones based on the ShowEQ map file data. It's very fast, and quite thorough. I want to know if there's any point in my posting it
here, since it's about 20K (although I think it will drop back down to 15-16k once I cut the obsolete routines back out). All the documentation is included in the top of the file, and while it CAN readily crank out a map that's a one-pixel-per-foot ratio, I've added some maximums that the user can change
which will make it automatically scale maps down if they are going to turn out above a certain size.

I have absolutely *no* intention of teaching
people how to use perl scripts, install perl
modules, etc. So much like ShowEQ my attitude is "if you can't figure it out then you probably don't need to be using it." Don't post back here asking me how to use it.

Provided one knows a teeny bit about invoking scripts and reading english, it shouldn't be hard at all.

If this doesn't become a huge pain in the butt, a quick modification might be able to put the line comment field to work--like making anything named "path" automatically draw as a ten foot-wide swath instead of a pixel-wide line.

Someone speak up if they want this program and I'll post it here... otherwise I'm not going to invite myself to get a bunch of silly questions from people who don't really care if they have accurate graphical maps outside of ShowEQ or not...
