Auto-Responder

Let's say you want people to be able to send mail to a specific address at your domain and get an automated message in return. EnterAct, for example, has an auto-responder set up for mail sent to info@enteract.com.

Another practical use for auto-responders can be if you are going to be out of town or if you are unable to respond to your e-mail. You can set up an auto-responder to answer your mail for you with a generic message stating that you are out of town or whatever you like.

In order to use procmail, you need to create a few files. You will need to create the message your auto-responder will send, the .forward file and the .procmailrc file. If you'd like further information about procmail and other things it can do for you, check out this page.

info.txt

Create a file in your root directory and call it whatever you like. Lets call it info.txt for argument's sake. Write your auto-response in this file. The following commands will satisfy these requirements. From your root directory, at the UNIX prompt, type:

cd ~
pico info.txt

Write your message and exit pico with control-x.

.forward

This file is typically used for forwarding mail to some address. In the case of auto-responders, we are going to use it in conjunction with procmail to send a specific message to whoever requests it. Follow these next few steps to create this file.

  1. Type pico .forward.
  2. Enter the following line into the editor:
    "|IFS=' ' && exec /usr/local/bin/procmail -f- || exit 75 #username"
      (Remember to change username to your own username.)
  3. Exit Pico with control-x.
Now you will need to create your .procmailrc file.

.procmailrc

Follow these next few steps to create this file. If you'd like a text version of this that you can easily cut and paste, click here.

  1. Type pico .procmailrc.
  2. Enter the following into the editor:
    PATH=$HOME/bin:/usr/bin:/usr/ucb:/bin:/usr/local/bin:.
    VERBOSE=off
    MAILDIR=$HOME/mail
     DOMAIN=yourdomain.com
      (Change yourdomain.com to your domain name or to enteract.com if you don't have a domain name.)
    LOGNAME=the username of the autoreply (in this case, info)
    LOGFILE=$HOME/procmail.log
    SENDMAIL=/usr/sbin/sendmail

    :0 c
    * !^FROM_DAEMON
    * !^X-Loop: info@yourdomain.com
    * ^TOinfo. *@.*yourdomain.com
    | (formail -r -A"X-Loop:info@yourdomain.com" ; cat $HOME/info.txt ) | $SENDMAIL -t -finfo@yourdomain.com

      (Change info.txt to the filename of the message you will be sending via the auto-responder.)
