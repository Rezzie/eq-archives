How do I get my web stats?


Once upon a time, you could search through the main httpd access_log to find the entries corresponding to your page. However, to protect the privacy of our fellow customers, we've decided to do this for you. (It also saves you the bother).

If you'd like to see how many hits your home page is getting, check out the directory ~/web_logs (or ~/vhost_yourdomain.com/http-logs/ if you are a CompleteAct customer).

Each SoloAct customer with a home page gets a weekly statistics summary of all access to their web page, as well as all raw log files apropos their page. The files will appear in your ~/web_logs directory around 3:30am on Saturday mornings if there was any traffic to your site in the past week.

CompleteAct customers get stats pages generated with the frequency that they have specified that they wish their log files to roll. If you haven't specified that you want your web logs to roll, no stats will be generated. You specify that you want your CompleteAct stats to roll by placing a file called .roll in your ~/vhost_yourdomain.com/http-logs directory, containing either the word daily, weekly or monthly, depending on how frequently you want to logs to roll.

If people have accessed your site before you created the .roll file, the statistics for those hits will not be generated on the first, or any subsequent, time that Analog is run. Only hits since creating the .roll file will show up in your statistics. You can, however, run the following command in order to generate statistics on the access.log file that was kept prior to your creating the .roll file:

analog -n"yourdomain.com" <logfile name> > stats.html

Replace <logfile name> with the name of your log; probably access.log. This will generate statistics on your access.log file and put them in a file called stats.html. Move this file to your http-docs directory with the following command:

mv stats.html ~username/vhost_domain.com/http-docs

You can then view this log by entering the following URL in your web browser:

www.yourdomain.com/stats.html

Your raw access log file will look something like this: 

dialin_6.enteract.com - - [12/Jul/1996:02:38:08 -0500] "GET /~foo/ HTTP/1.0" 200 2027
dialin_6.enteract.com - - [12/Jul/1996:07:34:50 -0500] "GET /~foo/foo.gif HTTP/1.0" 200 3500
gw.att.com - - [12/Jul/1996:07:57:35 -0500] "GET /~foo/foo.gif HTTP/1.0" 200 3500
The access_log lists every individual retrieval of a page in your web space. The requesting host, time, file request, status code and file size (respectively) are logged.

The file stats-mm-dd-yy.html (www-stats-mm-dd-yy.html, for CompleteAct customers) contains the results of running the log analyzer program "analog" on your access log. It is formatted in HTML. If you want to see it in your web browser, copy it to your public_html directory and change the permissions so that it is readable by everyone chmod a+r stats-<date>.html (or FTP it to your home computer) and fire it up in your web browser. stats-<date> will look something like this: 


[...some header material...]
Total completed requests: 23
Average completed requests per day: 7
Total failed requests: 1
Number of distinct hosts served: 6
Total data transferred: 135 455 bytes
Average data transferred per day: 43 981 bytes
[...Followed by listings of your more popular pages, and other fun stuff...]

If you would like more information of the statistics package we used to generate these reports, please see the analog home page.
