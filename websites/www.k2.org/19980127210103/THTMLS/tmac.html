Configuring MacTCP and/or TCP/IP

Configuring CONFIGPPP

The following assumes you are using FreePPP 1.0.5. (see 'net 'ware for availability)

HERE is FreePPP 2.5 beta (final April 14), but you're on your own with this one.

Under the "Apple Menu"
under "Control Panels"
  1. open "ConfigPPP" and enter or verify the following settings: on the main menu:

    port name: modem port
     idle timeout: none
     echo interval: off
     Disable Auto: unchecked
     Terminal wind: unchecked
     hangup: checked
     quiet mode checked
     long re-dial unchecked
     ppp server untitled

  2. Now click on "Config" and enter or verify the following settings:

    ppp server Enteract
     port speed 4 x modem speed
     flow control cts&rts(dtr)
     tone dial checked
     phone number enteract's phone
     modem init nothing
     modem timeout 90 secs

  3. click on "Connect Script"

    timeout: 40 seconds
    everything should be set to "out"

  4. click on "authentification"

    enter YOUR username and password
    timeout - 3 seconds works on a 28.8 modem, slower modems might require more time...

  5. click on "LCP options"

    click default settings (again, longer timeouts on slower modems)
  6. click on "IPCP options"

    click default settings (again, longer timeouts on slower modems)

  7. click on "done"

MacTCP

    under control panels
    under MacTCP

    NOTE (if there is no MacTCP, but there IS a TCP/IP control panel, see below)

    select PPP

    click More...

    obtain address
    click manually
    set Gateway address to 0.0.0.0
    click Server

    Class: C
    Subnet mask: 255.255.255.0
    (set by moving weird slider if necessary)

    Domain: enteract.com. (lowercase)
    no secondary domain names

    IP Address: 206.54.252.1
    no secondary IP addresses

    Default: Checked

    click OK

    close MacTCP

TCP/IP

    For TCP/IP (mutually exclusive with MacTCP - if both appear, you've screwed up. Going to your extensions manager under control panels and turning one off MIGHT solve the problem)
    Connect via: MacPPP
    Configure: using PPP Server
    Name server: 206.54.252.1
    Search domain: enteract.com

    close TCP/IP

    PLEASE NOTE:

    According to the powers that be, as of 3/26/96, if you're running under system 7.5.3, and using Open Transport 1.1 (TCP/IP control panel), you should:

    1. Open the TCP/IP Control Panel
    2. Under Edit, Choose "Advanced" user mode
    3. Click on "Options", and UN-CHECK "Load only when necessary"
    4. Save this configuration
    5. On the ConfigPPP control panel, CHECK "Disable AutoConnect" (to prevent your computer from automatically trying to connect when you start up)

    For the technically minded, this (hopefully) eliminates the memory leak and heap fragmentation by loaded OT under the system instead of on top of it.

    At this point, you may have to restart your mac, and you should be golden.


TO LOGIN:
  1. open ConfigPPP Click OPEN

  2. (Assuming PPP has "Connected successfully") Launch Netscape (or whatever)

  3. When finished surfing, quit netscape

  4. open ConfigPPP (if you closed the window) Click CLOSE
NOTE: On Macs with Open Transport 1.0.8, you can only connect once, then you have to reboot.
