    Winsock.dll

    If a you have several applications which use your modem (e.g. AOL or Trumpet Winsock), chances are very good that you will have more than one instance of winsock.dll on your hard drive. If this is the case, it could cause problems for you when you try to connect to EnterAct. If you find that it looks like you are connected, but then you are unable go to any web page, retrieve mail, or read news, it is possible that you have more than one of these files and they are conflicting.

    To solve this, do a search for winsock.dll from your file manager or from within DOS with the C:\>dir /s winsock.dll command. You should have one in your Trumpet Winsock directory. Any others should be renamed to winsock.old

    Manual Login

    Most often when using Trumpet, login problems are a result of trying to log in automatically. Earlier versions of Trumpet do not include a usable login.cmd for EnterAct. We do have a custom login.cmd at our public ftp site ( ftp.enteract.com), but we haven't found that replacing it fixes login problems. Our recommendation for Trumpet 2.x users is to either download Trumpet 3.0, which is very easy to login automatically with, or to dial in manually. You simply need to go to the dialer pull down menu and choose manual l ogin. After you dial, you will be prompted for their username and password. After entering those, you will see garbage stream across your screen. Hit your Esc key to finish your PPP connection.

    Netmask & Gateway

    These two values must be set correctly (255.255.255.0 for Netmask and 0.0.0.0 for Gateway) in order for Trumpet to connect to us properly. In o rder to change these values, you will have to deselect the Internal PPP check box. When you've changed the appropriate values, check Internal PPP again.

    MSS and MTU

    These values should initially be set at 1024 and 1500. If you have problems with slow connects, try lowering the MTU to 576. It may help.

    Comm Port

    If your modem does not respond at all, or if when manually dialing nothing you type appears, check the com port (it will be called slip port in Trumpet). Make sure it is set correctly. If you don't know which port is the correct one, keep trying different ports until you get the right one.

    Baud Rates, Init Strings, and Comm Overrun Errors

    Many times you will get com overruns with Trumpet Winsock. There are a few things that can fix this. 1) Make sure MSS is set to 1024. 2) Make sure that the baud rate is not set too high (no higher than 57,600 for 28.8 modems and no higher than 38,400 for 14.4). 3) Make the init string ATZ. I've noticed that some people who are using AT&F 1 in Trumpet are getting comm overruns. When they change it, the error messages go away.

    Login Woes and PAP (Password Authentication Protocol)

    This should not be selected for Manual logins. It can be found under File/PPP Options and you will not authenticate if it is selected. Also verify that the username and password are being typed in correctly (logins are lowercase and passwords are case sensitive).

    Try Different Phone Numbers

    A good thing to try if things don't seem to be working right (you aren't authenticating, or you get an IP address but can't ping) is to try a different phone number on a different system. This will often fix the problem.

    If All Else Fails...

    Exit trumpet, restart Windows, and try again. Can't hurt, can it?
