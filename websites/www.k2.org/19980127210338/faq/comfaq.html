Terminal Programs

Setup

All communication programs, whether it be Terminal, Hyperterminal, ProComm, Telix, or RipTerm, are essentially configured in the same way. Make sure the following settings are correctly entered in your software.

  1. Parity = None
  2. Data Bits = 8
  3. Stop Bits = 1
  4. Flow Control = Hardware

Terminal Emulation

This should be set to either VT100 or VT102. If it is set incorrectly (ANSI for example), your screen will not look right and the formatting will be all wrong. Simply changing the Terminal Emulation makes all the difference.

Com Port

If you are having trouble dialing out, verify that the Com Port you are using is the correct one. If you are unsure, try Com1 first and then Com2, and so on. If the wrong com port is selected, the modem will not dial.

Baud Rate

Try lowering your baud rate. With some modems, if your baud rate is too high, you may not connect well, or even at all. If you have a 28,800 modem, try setting your baud rate to 57,600 bps. And if you have a 14,400 modem, try setting it to 38,400. You can always try raising it up to 115,200 for 28.8 modems or 57,600 for 14,400 modems to see if you can achieve faster speeds.

Bad Login?

Make sure you log in with an R in front of your username. The username must be lower case. The password is case sensitive and must be typed exactly as it was created. Remember, when dialing into the shell with a communication program, you will be prompted for your password twice. This is normal. If you mis-type your password the second time, you will have to re-enter your username without the R. This is because the first password logs you into the terminal server (which is why the R is needed. Without it, the terminal server would think you were trying to obtain a PPP connection). The second password logs you into the UNIX system (usernames must be all lower case, so the R would result in a bad login.)
