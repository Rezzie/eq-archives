Winsock.dll

If you have several applications which use your modem (e.g. AOL or Trumpet Winsock), chances are very good that you will have more than one instance of winsock.dll on your hard drive. If this is the case, it could cause problems for you when you try to connect to EnterAct. If you find that it looks like you connect, but then cannot go to any web page, retrieve mail, or read news, it is possible that you have more than one of these files and they are conflicting.

To solve this, do a search for winsock.dll from your file manager or from within DOS with the C:\>dir /s winsock.dll command. You should have one in your Netscape directory. Any others should be renamed to winsock.old.

Baud Rate

Try lowering your baud rate. With some modems, if your baud rate is too high, you may not connect well, or even at all. If you have a 28,800 modem, try setting your baud rate to 57,600 bps. And if you have a 14,400 modem, try setting it to 38,400. You can always try raising it up to 115,200 for 28.8 modems or 57,600 for 14,400 modems to see if you can achieve faster speeds.

Properties

  1. All of the configuration settings for Netscape P.E. can be found by double clicking the Connection icon and going to the Properties button.
  2. If you are switching from another provider, however, it may be easier to edit the myaccoun.sr file. (Note: this file may actually be called something else. You may have isp.sr, enteract.sr, internet.sr, or something else. The netscape.sr and default.sr should be left alone.)
  3. To edit this file, go into DOS.
  4. Type: cd \netscape\dialer
  5. Type: dir *.sr. This will give you a listing of your .sr files.
  6. Type: edit myaccount.sr (again, replace myaccount.sr with whatever your .sr file is called.)

Here is what that file should look like: 

[Location:Home]
AreaCode= Your Area Code
Version=2.0
DialAsLongDistance=Yes
DialAreaCode=Yes (This might be No if you dial within the same area code)
Phone= Our Local Phone Number

[Dial-In Configuration]
CurrentLocation=Home
Name= Your Username
SPassword= This will have an encrypted password.  If you have problems 
     logging in, make this field blank.
Phone= Our Local Phone Number
SiteName=EnterAct

[IP]
DomainName=enteract.com
DNSAddress=207.229.143.1
DNSAddress2=207.229.143.2
IPAddress=0.0.0.0
Enabled=Yes

[Script]
ScriptEnabled=No
ScriptTerminal=No
ScriptFileName=C:\NETSCAPE\DIALER\EnterAct.SCP

[NetBEUI]
Enabled=No

[IPX]
Enabled=No

[Security]
SecurityDevice=Yes
SecurityEcho=No

[Windows]
Connected=442,0
Main=0,-1

[Locations]
location0=Home

Are the DNS Addresses Correct?

DNS is a service that translates numeric Internet addresses into easily-remembered hostnames ("enteract.com", for instance). In order for this to work, your Internet software needs to know the numeric address of EnterAct's domain name (DNS) servers.

If your DNS servers are specified incorrectly (or not at all), you'll be unable to locate sites by their names, and will only be able to talk to sites using their numeric addresses. If one of your servers is right and any others are wrong, it will take a long time to locate sites by their names. For the best possible results, please make sure that you've set up your software to use:

Primary: 207.229.143.1
Secondary: 207.229.143.2

Microsoft Office in the Startup Group

Netscape tech support claims that having Microsoft Office in the startup group can prevent Netscape from working. While we have no idea why, we're inclined to believe them.
