Patch Message: 11-16-00
[6 Comments] - Aurum @ 5:40 AM PST
This message went up yesterday along with the follow-up patch that was implemented:

    November 16th, 8:00am
    ------------------------------

    *Follow-up Patch*

    Today's follow-up patch addresses the following issues from yesterday's patch:

    - The spell Torpor received an unintentional reduction in effectiveness
    - Illumination in game was not as bright as it should have been.
    - Pet using classes unintentionally gained the ability to summon multiple pets.
    - Bard songs were reacting poorly with invisibility/hide/sneak.
    - Hitpoints were being reported incorrectly to clients.
    - Changing resolutions would cause some to crash.

    In addition, the following changes were left out of yesterday's patch message:

    - Fixed a bug that would cause some to not be able to cast after being feared.
    - Data for group members and targets will be updated more frequently. This change should reduce complaints of warping and slow hitpoint and other updates.
    - Bards received increases in the skill cap for their instrument skills.
    - Changed the default combo for casting spells from CTRL-1:8 to ALT-1:8, as CTRL was interfering with players ability to strafe.

    Thank you for your continued patience.

    - The EverQuest Team

Technical Support Infobits
[10 Comments] - Aurum @ 3:24 PM PST
Abashi hit the Technical Support board and posted the following messages:

  • Sound Related Problems Temporary Workaround
    If you are having problems playing due to a sound-related issue, and just want to get in game in the meantime before working through the problem, you can put a file in your EverQuest directory named "NOSOUND.TXT". The contents of the file are not important. This will turn off sound and should eliminate any sound related problems.

    When troubleshooting to find out if your problems are sound or video related, it would be helpful if you would try this just to see prior to contacting technical support. It always helps to know whether we are dealing with sound or video problems.

  • Casting Spells Using Hotkeys
    Prior to yesterday's patch, you could cast memorized spells using CTRL+#. The default is now ALT+#, due to some conflicts with strafing and such.

    You can change it back to CTRL if you prefer that by customizing your keys via the options/keyboard menu in game.

Direct X Clarification
[31 Comments] - Aurum @ 7:47 PM PST
Clarifying the Direct X issue, Alan wrote and posted a new board post on the Gameplay Board:

    It seems that there might have been some confusion about DirectX. Here's the story.

    EverQuest is now set up to run with DirectX 7. However, Microsoft recently came up with DirectX 8. Versions of DirectX have been backwards compatible historically. But we have not had a chance to complete testing with DirectX 8. For this reason we highly recommend that you use DirectX 7 with EverQuest.

    HOWEVER, because there are some changes between DirectX 7 and 8, some of your video or sound drivers may not run properly with DirectX 8 installed. These problems should be fixed when your sound or video card manufacturer updates their drivers for DirectX 8.

    The same is true about earlier versions of DirectX. Because EverQuest is now utilizing the power of DirectX 7, you will most certainly need to install the newest drivers for your sound and video cards (and do so AFTER you install DirectX 7 or 7.0a – for EverQuest there is no different between 7 and 7.0a).

    WE RECOMMEND DIRECTX 7 or 7.0a for use with EverQuest.

    It seems that there is no easy way to uninstall DirectX 8. This is also beyond our control. But if we find something that will aid with this, we'll certainly let you know.

    I'm sorry if there has been any confusion.

    Alan
