There is something called "process" in the development of a program. Process is a set of procedures, habits, and "processes" that are designed to ensure that the development goes smoothly, has few errors, and is able to handle any problems that arise.

Most process is lumped under: code control (making sure you don't overwrite newer code), design control (making sure that your design elements don't override each other), test control (making sure you properly and thoroughly test for bugs and problems), document control (making sure you document your development properly so test control can do it's job, and so future developments on the same program can be done without overwriting, making mistakes, or re-writing something already done), and of course management (the process of making sure everything is covered).

Verant has very little of these. Case in point?

"Ooops. We accidentally overwrote the new code in the last patch." (code control violation)

"Oops. We didn't think about that." (design control violation)

"We didn't realize that the new levitation changes would affect bard songs, despite the fact that their object is derived from the magic objects" (document control violation)

"You can't expect GZ to remember everything!" (nope, thats what process is for...management violation)
