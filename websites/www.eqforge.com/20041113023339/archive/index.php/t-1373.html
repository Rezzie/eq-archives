I prefer to use the in game editor initially and then only after I have planned the entire tech tree on paper. Some advances trigger different things like new city icons or making barracks obsolete. Also, certain improvement groups do not allow the underlying tech to be changed (like temple). So its not a good idea to change the tree and wait to see what happens.

But anyway in this case, it looks to me like everything's preq is either nil or no. (What kinda tech tree is this???)Just make the preq for future tech nil and check that its attractiveness is still one. That should stop the crash. If not, as willem says, its probably a spelling error.

Last but not least, a word of advice. You should work on the sounds first, then the units and then the tech tree in that order.

Hmm, it´s hard to find here. I had the same problem one time.
My solution: take a new rules.txt (standard civ) and paste the first ten techs over the existing techs, then start the scn. If it runs, paste the next ten techs etc. If an error occurs, you know that the problem must be in the ten lines you´ve pasted before the last scn start. It´s a bit work, but I found my bug easily with that method...

BTW, make sure that you have exact the same number of techs as in the standard game. One tech more or less, perhaps due to the pasting, and your scn will crash...<IMG SRC="http://forums.civfanatics.com/ubb/wink.gif" border=0>
