xeven's offer to help is commendable but I suggest that you still do the tree yourself (with his help) because you have to do the entire tree in your head anyway and you need to somehow pass that info to xeven to input. And you will keep changing it (believe me) and you can't get xeven to amend it each time.

The mainstream way to do tribe specific trees is to use No No techs. If you hae never done a tree before, I do not recommend using this menthod on your first try.

Instead, just make the prerequisite of the first tech of each tribe's tech branch to be the last tech on a general branch available to all races. Then edit technologies using the cheat menu and give each tribe its starting tech.

Good luck.

kobayashi
May 28, 2001, 10:06 PM
Not too much into using barbarians myself. When I need barbarians, I usually designate a tribe to be barbarians to control them better because real barbarians are completely random unless they have a city.

But if you are all fired up to use real barbarians, this is what I know. Start the original game and choose the barbarian setting you want before saving as a scenario the first time. Any barbarian which captures a city will force the city to keep producing itself. You can make barbarians by manually cheating but not through events.

The rules for which barbarians appear are well set out in Will Keenan's Barbarian Paper which is the authority on all matters barbarian.
http://sleague.apolyton.net/Guides/sldt_barbtechs.shtml

Generally it depends on what techs have been discovered. You must keep this in mind when making the tech tree.
