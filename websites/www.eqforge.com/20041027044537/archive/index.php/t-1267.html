Just a few words from the point of the FW version.

The readme says to overwrite the pedia and labels files in the main civ directory. That is not neccessary (and also doesn't work the way it is now).

For FW to use a labels.txt file generated from MGE, the first number below @labels must be changed to 606 from 800+ at the top. This has something to do with how many lines of command are to be used (the MGE version has got lots of extra labels starting from player 1, player 2.....). If FW gets labels for items which don't exist, it will crash.

That being the case, you should either have a seperate labels.txt or tell the guy to edit the file after he downloads it.

As far as pedia.txt is concerned, it seemed to work fine when it was put into the scenario folder.

I've only played a few turns and so far the terrain look is terrific and the units (like the fortress that pops up) are great. I look forward to playing the scenario this weekend.
