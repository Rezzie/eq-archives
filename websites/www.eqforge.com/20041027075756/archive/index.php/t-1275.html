Most common crash causes...

Graphic improvements or Rules.txt improvements? In that second case, made backup copies and start changin' names one at time... I know that's annoying but sometimes help.

Another cause can be Title.gif. If this file has wrong palette, your scen will crash suddenly.

Wrong spaces, shield values or attack/defensive hit points misplaced also causes scen to crash.

Try to check all of these points.

Generally there is an error code for the crash. If it is giving you a file start up error such as in "MODULE.DB" you can bet there is a problem with your rules.txt

Otherwise, it also sounds like you may have a graphics crash where the units.gif is not saved in the correct format...also, as Prometeus said, the title.gif can cause this problem too.

Make sure that you don't have too many values out of their normal parameters such as mentioned before, but that also goes for some req'd techs too...

Give us more detail if none of these suggestions work....
