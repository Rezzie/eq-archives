I of course like everyone else cannot decypher the error codes...however, if you can tell what you did before the crash, that would help.

As some general advice, crashes fall into three types:

1. Oversized events file

2. Tech tree that doesn't stretch from nil to future tech

3. Editing with the rules and other files manually (this is most likely what happened). Best thing to do is to stick to using the scenario editing tools (i.e. in the program itself) when you start off. The only time you really need to do manual editing is when you introduce No-No techs.
