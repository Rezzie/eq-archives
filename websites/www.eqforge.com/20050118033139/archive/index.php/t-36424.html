Usually that is a miscalculation. You processor clock runs relatively stable, with peak only in the 0.x% range. Anything more would cause you programs to misbehave or crash regularly, as certain program cycles couldn´t behave as intended, especially under a multi-thread environment like Windoze. :lol:

Whatever program you use is always a software calculation of the CPU clock, it times the speed by counting and comparing high-level programming cycles. Since the code the CPU understands is entirely different, this is not really a valid way of testing the speed.

The quartz that creates the system clock and the CPU clock runs with a precision in the millionths.

More importantly, as you can read from the numbers, your input voltage level did not change. But this is necessary to change the internal clock speed of a CPU.

So, it was probably a simple software glitch. Also note that you only had it running for a few minutes, this does not exactly account for a sufficient number of measurement points.
