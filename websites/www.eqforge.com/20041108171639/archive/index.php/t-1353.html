This may be an old tip, but anyway still very useful for new designers like myself. I thought this might be worth the mentioning again :

In order to make the Civiliopedia work with your new tech tree and city improvements, units, wonders etc., you must remove the file "Get info.exe" from the pedia folder.

A good way to edit city improvements and wonders etc, is to alter the "pedia.txt" in the civ folder, as you design the tech tree. It makes one aware of many things that need to be accustomed to the scenario, regarding what effects improvements and wonders have, in combination with civ advances.

The version I have is the original Civ2 -however I don't know which version of the original that is.

But there is both a "Pedia" folder (which is installed with the full installation), that contains the A-Z index of the game, as well as detailed descriptions, pictures (supposedly the ".apr"-files), etc of the civ advances, units etc, AND a "pedia.txt".

The "pedia.txt" is placed in the main folder of the game, and can easily be altered.
The thing that buggered me, was that I wanted the Civilopedia to show how my new tech tree worked out. If you remove the "Getinfo.exe" program from the Pedia folder, it seems like CIV2 gets the information from "pedia.txt" -otherwise it seems to get it from the pedia folder!

I don't know how this works with TOT or other versions of the game. I suppose with some other versions, the game supremely gets the info from "pedia.txt" and you won't have this problem.

The catch with removing "getinfo.exe", is that you no longer can get instant info on the terrain, it seems. But that is a minor problem, I think.
