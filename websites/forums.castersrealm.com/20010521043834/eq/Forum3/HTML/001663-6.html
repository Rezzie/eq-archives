wow, thanks for all the great comments. Id like to answer a few questions here:

1>screen resolutions:
Well, my idea is that this interface woudl get rid of bitmap based UI elements and go purely with vector(like what adobe illustrator, or even windows works with) graphics. They are resolution independant, and a good engine can take up a LOT less processor cycles =)

2>small UI elements:
see number 1. if you use vector graphics, you can esily scale w/o a loss in quality or readability(unlike bitmap graphics). in most cases, the quality of the object stays, so you could conceivably switch resolutions and keep the UI in the same aspect ratio.

3>moveable/removeable UI elements:
this can be implemented on any UI. Just takes some forethought to do it.

4>buff slots:
only 15 buffs alowed on you at any givin time =(

5>gothic look and feel:
I personally dont care for bits of UI that serve no purpose, but it IS an adventure game. Ill see if i can mock something up(these kinds of elements dont work well with vector based UI's, because they take up a lot of memory, and dont scale well)
