quote:
Originally posted by Trinket:
Root is usually not an option since FGs tend to summon

Root is a perfect option for summoning mobs. A rooted NPC does NOT summon.

This is because summoning was originally implemented as anti-exploit code as follows:

if an NPC can legally move, but cannot reach its hated target, it will summon the target.

A rooted NPC cannot legally move, and therefore will not summon.

Watch closely the next time you fight level 51+ NPCs that summon. Root the one being fought, then note that the same tick you see "Your Root spell has worn off", the NPC will start summoning again.


Root is absolutely perfect for handling unmezzable NPCs of any level, provided they aren't immune to magic in the first place.

-- Hika

Since my higher level character is a Shaman ( and thus i have to root/park when there is no enchanter), this is what i have found :

When a summoning class mob is NOT damaged in any way, and rooted (not druid root since this does damage), they dont summon. If they are damaged at ALL ( ie damage shields, Melee, DD, Dot Etc) they will summon if they cant whack someone in melee range.

So..

If your root parking a mob as a form of crowd control, make sure your pullers dont have on any form of DS, as then all the mobs will be damaged (and then start summoning you.

You can also root the target your tank is on (gonna piss off your snarer tho) to keep the mob from summoning YOU, the fragile enchanter, regardless of how much hate it has. If a mob is rooted, it effectively hates whoever is closest to it, untill the root breaks anyway. As long as you have a mob rooted, and melee types are smashing it, it wont summon.

I have never seen a root stick to a dragon / god type mob so i dont know about them =)
