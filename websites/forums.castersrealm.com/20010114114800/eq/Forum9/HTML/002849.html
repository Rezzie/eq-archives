Oh the pain of it all Fryg... its been a most traumatic 5 days since the patch for me. I had the same problem since patch... latest of all drivers... but everytime I went to mouselook, move or save... the Hard Drive on my PC would whirl on and freeze everything until it had finished doing whatever it was doing.

But, after searching many many boards I found a course of action that has got me running smoothly again. In brief, there is something in your machines startup config that's conflicting with the newer version of EQ Client/Direct X. In order to find out what this is, try this... it worked for me... given I was running under a version of Win 9x, e.g. 95, 98. I can't guarantee its going to work for you, but its worth a try.

Go to the Start button -> Run -> Type in MSCONFIG then press enter. Then go to the Startup Tab and turn everything off, except the Systray option. You should find that the game runs a whole lot better now.

What you need to do next, go back and add the items in the Startup list back in one by one, rebooting every time you do it. When it starts to go slow again, whatever you just added is causing the problem.

Good luck, I hope it helps for you.

Go to the Start button -> Run -> Type in MSCONFIG then press enter. Then go to the Startup Tab and turn everything off, except the Systray option. You should find that the game runs a whole lot better now.

If that is happening, then the real problem is that your system is loading a lot of crap when you boot up. For a stable system overall, you should try to have the LEAST number of resident programs possible. Virus scanners are the worst culprits here -- turn them all off.

If you use an Nvidia video card (you should ), check that you are using the Nvidia reference drivers (Detonator 3) from nvidia.com. Some video card manufacturers have less-than-wonderful drivers, whereas the reference drivers are nearly bulletproof, and most likely faster. Also, some video card manufacturers load extraneous resident programs that make your system less stable.

Fryg, know the feeling well.. and with it relearned all the prime actions for clean install of any driver.

You've heard the top suggestion, install DirecX then Sound Card, then Video Drivers...

My personal take after going around with a sat modem and learning what the tech support crew said to do; here's some ideas.

Uninstall your specific video card drivers, by loading the PCI VGA Standard Driver, reboot and then remove all programs associated with your Vid card from the Add/Remove programs.

AT this point, ready to install the drivers? No, not really, you'll want to bring up the Task Manager (Win 98 cntrl+alt+delete one time) And turn everything off - "End Task" but systray and explorer. (this gives you as close to a clean boot as we'll get.)

Beware of the latest video drivers. As nice as it is believe they only improve, I can tell you for a fact the ELSA GeForce 2 set xxxx.150 suck to EQ right now. If you had no problems with your old drivers to EQ and you've updated to DX 7.0, 70a, or 8 try the above install suggestion regards remove and reinstall after the DX update.

Now, for the fun part... For some reason, and extrodinarly hard to understand, VI's work with Sound Blaster EAX appears to be behind many other software writers. This is where you have to run "C:\Program Files\directx\setup\DxDiag.lnk" (or where ever you might find it on your machine). This brings up the DirectX Diagnostic Tool.

On the Tabs up top you'll see one for Sound, selecting it will give you access to the DirectX Features, Hardware Sound Acceleration Level slider bar.

If you are at "Full Accelertion" VI has indicated one way to get rid of the momentary freezes in game is to back off one notch. (Standard Acceleration)

IF all the above doesn't get it for you other people have pointed out, and it worked for me once as well, to go to the EQ files director and remove the files associated with the particular zone you have troubles with. On your next log in to the game the patch server will note the missing files and push them back out. *Of course, the reload time might be substantial.*

This appears to clear any sort of corruption that might have occured to the data - even if a "Thorough File Check" doesn't turn anything up.

When your character is saved you're doing a small write to your hard drive and (guessing here) sending some packets to the server. If the game pauses due to the overhead of a disc write, there could be a number of causes. This is not an exhaustive list, but you can try:
- Turn off Fast File Find. Windows keeps a table of the most recently accesed files, their SIZE, and the physical location of their first sector on the hard drive. This is one of Microsoft's workaround for their lousy FAT on 9x systems. Anyway, any time you modify a file, this table is also updated and it can add delay. Whatever, turn off Fast File Find in your control panel.
- While in Control Panel, look in System/Performance/File System/Troubleshooting and make sure all the little boxes are unchecked.
- While in Control Panel, look in System/Performance/File System/Hard Disk and set the Typical Role as a Network Server. This will give you a deeper write cache.
- While in Control Panel, look in System/Performance/Virtual Memory and check the box that says "let me specify my own settings". Create a swap file on your fastest hard drive with a size maybe 2x your RAM, but no larger than 200 MB in any case. The size is not critical. This swap space will be only marginally faster than normal disc space (because it's continuous) but that's not the point. The problem with letting Windows manage virtual memory size is that it's continually resizing the swap file to meet current demand, and the resizing takes time.
- Have you made any hardware changes lately? If you have IDE drives, put the hard drive(s) on one channel and DVD/CD-ROM/ZIP on another channel. IDE electronics are not smart enough to negotiate bus speeds on a per-device basis, and so will always run the bus at the speed of the slowest device. Putting your Ultra/66 capable hard drive on the same channel as your PIO CD-ROM is a bad idea.

You don't say if this problem has been with you forever or is as a result of the latest patch. I honestly don't know why upgrading to DX7 would cause this sort of problem. DirectX does not modify file system settings, afaik.

You should also do a full defrag on the slow system. If your character file is split the save will take longer, especially if the game does a confirming read after the write.

Good Luck
