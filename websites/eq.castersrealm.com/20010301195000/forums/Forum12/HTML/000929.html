Remember a few weeks ago when Togor's was changed in the spell database to a max of 60% slowdown at level 50 (from the previous 65%), but Abashi assured everyone there's really no nerf, and that they're just moving it from the data file to server code?

Well tonight I checked the spell data file. It still shows the 60% figure. Then I logged on and tested it -- same method as I used several weeks ago. Because I was only testing one spell, I even ran the test for a little longer, for about 3 minutes with a togor's refresh in between.

My result was a modified attack speed of 39.7% (60.3% slowdown) using one calculation method and 42% (58% slowdown) using another calculation method (I believe the true figure lies in between). Sure sounds like the figure in the spell data file is accurate.

Perhaps someone else could test it out to verify, but from this test it appears that no server-side code changes were made at all - just the total effect of the spell reduced.

[Edit] I had another thought - maybe the spell never really had an effect of 65% slowdown. In my original test the result was a modified attack speed of 38.2% (61.8% slowdown) using method 1 and 37.7% (62.3% slowdown) using method 2, not the 65% that EQ Caster pulled from the spell data file. Since the other empirical figures matched the spell data figures so neatly -- almost exactly -- I believed that the togor's result was skewed (due to lag or testing wierdness) towards making it look less effective than it was. Perhaps it was really skewed the other way and the spell was always capped around 60%. That would also reconcile the result with Abashi's statement, sort of.

Anyway, enough rambling. The only thing that's pretty clear is that you're not getting 65% out of this spell, unless someone wants to run their own test and prove me wrong.
