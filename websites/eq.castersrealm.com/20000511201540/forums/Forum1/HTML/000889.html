Abashi:
I think it would be really nice if GM's had the ability to monitor what software was running on the client's computer.

This is really a dead horse Abashi. Software companies have been asking for these type of changes for years, and every single time us gamers have rejected the idea.

Awww, come on, why not?

It simply doesn't fit into our "Vision" of what a game company should be capable of.

But we need to stop showeq users!

I would agree that the integrity of the Everquest world is something that should be protected both to preserve our enjoyment of the game, and to preserve your customer base (We certainly don't need the out of control cheating that was going on in Diablo I)

However, the problem simply boils down this. All of the showeq type cheats, use packet sniffing. And from an information theory standpoint, it is impossible to tell whether someone is sniffing packets. Plain and simple, the TCP/IP architecture simply allows no way to tell whether a packet was read on it's way to you.

You could try using encryption, but the fact of the matter is, somewhere inside the Everquest files, there is a key to that encryption (since the client needs to read it), and as long as the client has the ability to decrypt it, there is no way you can stop the user from finding that key and decrypting it himself.

But if we can see the processes on their machine, we can look for showeq!

This is ridiculous. First off, you've prevented such programs like this being run on the EQ machine, simply by disabling the Alt-Tab. You've forced all of these users to run these type of data displaying tools on other machines.

But let's say, for the sake of argument, that they do run showeq on the same windows box as eq. They'll immediately distribute a new version of showeq that shows up as "notepad.exe" on the task list. Gee, sure glad you violated all 100,000 of your user's privacy rights, just to force some programmer to change 8 characters.

Well, i don't care whether our users don't like it, or whether it is completely unethical, we'll simply put it in our EULA.

I hope you keep abusing the End User License Agreement like this. What's next? After you can monitor the processes on someone's computer, you demand that all EQ users buy some completely unrelated software package? And if they don't buy it and run it, then they can't play EQ? Unethical yes. Would it be legal if you put it in the EULA, very likely. What are we gunna do, stop playing Everquest? No, we're addicted, but if you keep playing that Ace, a bunch of us are gunna get fed up and go cold turkey.

The real meat of this is, showeq type cheat programs run on different computers in a network than EQ.

It is impossible to see what these computers are doing.

If Verant has found a way to detect that one of these computers is packet sniffing, then they've invented a new type of computer security product, and could easily dump EQ and start their own security consulting firm.

Didn't one of the patch messages say something to the effect that "ShowEQ" was not the biggest offender? Maybe they're looking for something running on the same machine? Not sure what it would do (since you couldn't see it unless is uses multimonitor APIs).

I write Win32 programs for a living (Visual C++, Win98/NT/2000), and I hope they don't do something that ONLY works on 98 and causes 2000 to fail. That's not too hard to do (use an API only in 98), and it would suck for people using 2000. Yeah, I know, they don't support 2000, but it works.
