Just Down the Street from the Hash Bar

Hey, a new server is on the way for the Euros who have such crappy connections to Verant's boxes in the United States. Next stop--Amsterdam:

We're opening a new server, the second EverQuest server to be located outside of the United States.
The new server is called Kane Bayle and it will be located in Amsterdam.

We will be opening the server as a test server for a while to help test network and connection issues, so we need to have a lot of people testing it out. You will be able to access the new server by running TestEverQuest.exe. Once you do Kane Bayle will be one of the servers on the Test Servers selection screen. Remember, though, that this is a Test server and all of the Test server rules apply.

Important note: Because this is a Test server, we will be wiping characters when we take the server Live.

Keeping that in mind, we want to encourage you to log into the server and play for a while. We really want to see how it holds up under a full server load, and for that we need players logged in and playing. Now is also a good time to try it out and see if Kane Bayle will provide you EverQuest with a better Internet connection. So please give it a try.

Thanks for playing EverQuest, and have fun on Kane Bayle!

- The EverQuest Team

Methinks the team that set up the new server has a massive case of the munchies right now. Let's just hope the Customs agents can still be bribed. "No way, bro... I have, like, no idea how that got into my suitcase..."

Please forgive our rather ham-fisted attempt to make the announcement of another Euro server into something even mildly interesting or worthy of repetition.

See How You Stack Up

A reader named Spytfyre tipped us off to a post by Verant's own Scott Hartsman over on the necro board concerning the possibility of stacking DoT spells. It's good reading, even if you know better than to play a necro:

DoT stacking has been mentioned enough times in the past (both distant and recent) as something we'd consider doing that I'd like to break character and explain in a bit of detail why it's harder than anyone had assumed.

(Please bear in mind - I'm still mostly the new guy. I've played forever, but I haven't been on this side of this particular discussion yet. :)

UsulDaNeriak wrote:

as i know BoD doesnt stack. both necs get healed, but one everwrites the others dmg. general opinion i have read so far was, BoD dmg doesnt stack. however, maybe i am wrong.

You are certainly not wrong. Quite the contrary - You couldn't be more correct.

Zarloth wrote:

I suspect the stacking code is a huge mess. It was made with only a few limited spell lines in mind, and now with all new spells and stuff, the limit is hit.

The observation that "It was made with only a few limited spell lines in mind..." is 100%, dead-on correct, with all of the associated difficulties that you might assume.

Please assume for a moment that some of those difficulties include, for example, an extremely limited amount of storage for each effect (buff/debuff), for each spell, for each PC, for each NPC, and so on.

And further, that those extents were excruciatingly painful to change.

Sskeli wrote:

Yes the "who" of the casting is kept track of...how do you think you get the 'your spell has worn off' message?

...which is an absolutely logical assumption, and the vast majority of the time, it's correct.

Unfortunately, there's one problem, and that's the problem that prevents us from just saying, "Yep! We can make this happen. Give me 3 days..."

What happens to the "Wears off" message when you log/zone out, then log/zone back in?

99% of the time, you don't get it.

Unfortunately, the caster's identity is a best-guess.

...and therein lies the problem. If we stacked DoTs using the same logic, every DoT in the game would become a zone-in/zone-out exploitfest.

Not that the majority of people would do such a thing. We can probably all agree, though, that writing to the majority in this case means that the minority will take undue advantage.

That said, there are other changes that we're making that will make this type of change easier. It just takes time to decode what's gone on over the years and make sure the bases are covered.

A cleaner, shinier EverQuest will be able to bring this back into the realm of the possible. In the meanwhile, the best that I can offer are my apologies if this is something that was promised that we haven't delivered on.

Hope that helps at least a little. :)

Hartsman is EverQuest's Technical Director, and he's developed a very positive online presence since taking the job. We give a big thumbs up to any Verant employee who actually goes to the players' home turf and communicates with them--especially somebody who isn't afraid of the obsessed and freaky necros.

My Pet, Do You Have a Second?

As the druids pointed out a few days ago (see, I told you we've been slacking), pet summon spells on Test server currently have a one-second cast time. Rich Waters commented about this change:

I wanted to see what the impact would be if pets were all fast casting, pretty much guaranteeing that a character could summon his pet almost anytime. There are concerns about things like chain-casting pets and the effect it could have on soloability for instance. The changes on Test should allow us to see the worst case.

I'm not sure whether this particular change will go live or not. I'll see how it looks, and continue experimenting with other aspects of pets as well.

Pet classes think this change is cool. Of course, they might be excited by the prospect of 'sploiting it to summon pets continuously one after another, but we'll give them the benefit of the doubt.
