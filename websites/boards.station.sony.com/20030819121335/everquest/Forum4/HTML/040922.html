Now Astral Guardians (a Bristlebane guild) has set up a section on their website to allow visitors easy access to the most current guild listing available.. for any participating guild.

This feature is now available to ANYONE from ANY GUILD on ANY server and the more the merrier! There's no charge for this or anything, it's free, free free!
When you play EQ next, press ALT-G to open your Guild Management window.
In the upper right corner you'll see a DUMP button.
Press this button and your guild listing will be exported into your EverQuest directory named Guild_Name.txt where Guild_Name is your guild's name, for example Astral_Guardians.txt.
Then visit the Astral Guardians guild text upload page at: http://AstralGuardians.com/custom/GuildRoster.asp
and upload your text file by clicking on the Browse button and selecting your guild listing file and pressing SUBMIT (be sure to select your server).
When it's been submitted you'll be sent to the guild listing page where you can search the guild listings.

Go on over and check it out! More information: http://AstralGuardians.com/thread-view.asp?threadid=917

Future updates include more character information, guild roster pages which you can link to or use a domain like: "roster.guildname.com" ie: roster.astralguardians.com which is a redirection to our guild listing which has a rather long URL: http://astralguardians.com/custom/disp.asp?Search_Guild=Astral%20Guardians&iTop=500&Order_Name=ASC

Yeah the site doesn't parse that information but of course everyone either edits the file or trusts me to not store the information.

Basically I read the Name, Level, Class, Rank and Last Seen Date then move to the next line because the Personal and public comments are troublesome if the comment contains a COMMA because evidently Sony didn't want to Quote qualify the output. A good exported file would look like this:
"Field","Field2","Something, Else",4/1/2003,15,"Last"

Not like this (which is what we get now - similar):
Field,Field2,Something, Else,4/1/2003,15,Last

Notice now it looks like 7 fields instead of 6. Anyway, that's why I dont bother with the comments/notes... just too troublesome to try to fix the output file.

For those who know code:
(iBinLen is the binary length of the file)

For iLoc = 1 to iBinLen
Select Case sRawData(iLoc)
Case chr(10)
If sRawData(iLoc-1) <> chr(13) Then
iRec = iRec + 1
Redim Preserve sFileData(5, iRec)
iField = 1
End If
Case chr(13)
If sRawData(iLoc-1) <> chr(13) Then
iRec = iRec + 1
Redim Preserve sFileData(5, iRec)
iField = 1
End If
Case ","
iField = iField + 1
Case Else
If iField <= 5 Then sFileData(iField, iRec) = sFileData(iField, iRec) & sRawData(iLoc)
End Select
Next


So you can see I only grab the first 5 fields... the rest is trash-canned.
