Station Member 		posted 08-21-2000 07:42 PM           
Arggg:

You are correct, I just used the Windows API calls for private profile strings, and yes I could have spent the time to add comments to the ini file. However, I knew that the moment the file was in place that people would make "eqclient.ini" file editor programs that would put an interface on it far better then adding comments to the ini file. I was proven right when 2 such programs were released the day the ini file went live, and now there are several more.

Also, I do plan so allow you to change the colors in game with the client. You will be able to use / commands and also navigate through the options menus to change the colors. I believe this has been stated at least twice before by me here on the boards. So the ini file is merely a configuration file for what you will be able to change in game. I could have easily chosen to make it a binary file in a proprietary format, but I chose the ini format because it was simpler to implement (good old Win32 API), and I knew that it could be edited easily (even if the keywords I chose were cryptic).

So I made the choice to implement the bare minimum in the ini file itself. If I had intended for the ini file to be the only way to change it I would have made it much more user friendly.

-Rowyl

Rowyl
Station Member 		posted 08-21-2000 07:53 PM           
I've already explained why there had to be color changes in the process of implementing the custom color feature. To recap, if I was to leave all the text messages there original color but be editable, I would have to have hundreds of user colors to cover every message and it would be pretty unweildy. I chose to group similar types of text together into classes. My original implementation didn't have enough text classes, so I have added several more. I apoligize that I didn't have more in the beginning, but hopefully after this week we can all put this behind us.

The 2 new classes I added for spells (others messages, and failure messages) are user 33 and 34, and since you current ini files don't have them you will use the defaults (failures in red, and others in a darker blue then your own spells). Any new classes I add will be above 34 so you will get defaults instead of what was already in your ini file (like what happened for 26 to 32).

I also plane to get get the "old default" colors and code them in as defaults so that you can delete your ini file and when it creates the new one it will be back to almost what it was before I did any changes.

-Rowyl
