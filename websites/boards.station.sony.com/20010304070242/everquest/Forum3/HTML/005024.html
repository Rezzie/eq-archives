I doubt it. The performance issue is the computer having to use a swap file on your hard drive instead of RAM. Takes extra time to read and write to the file that doesn't exist when it is pulling from memory. I would not set a size for the VM file, just let Windows use as much as it needs. One thing I would suggest if you are not going to increase the amount of RAM would be to cleanup and defrag your hard drive frequently. This will at least allow the drive to work at optimal performance, but it will still be much slower than RAM.

Just my 2cp.

No - if you had too little virtual memory, EQ would crash with an out of memory error. If it isn't crashing, it has enough virtual memory, and adding more would only slow it down.

However, SPECIFYING the size of virtual memory yourself rather than letting Windows manage it (which you must be doing already or you wouldn't be asking about the size) definitely speeds things up. Windows wastes a lot of time messing with the VM size if you let it manage it itself. It used to be 200 meg was required for EQ, with Velious it's more (I think I set mine at 250).
