Fact:
When a player deletes himself, all corpses associated to that person are also deleted. This reference is done while the player is offline since you can't delete your character from inside the game proper. That means that there is already a function in the game to keep track of/find all of a player's corpses, offline or not.

Question:
If there is code to determine who a corpse belongs to already, how can it be too hard to reference the owner of the corpse's faction?
--

I won't pretend to know how your player database is structured, but I would think that each server must have some kind of database which contains every character on it. Regardless of the database structure, there has to be one or more Primary Keys to allow for character lookup.

Let's assume the Primary Key for a character record is the character's name (since they must be unique). Each corpse has the character's name on it. This means that you have enough information to lookup a character record and then extract the faction information from there. From a programming standpoint, any properly created database would be able to do this.

Let's say that the Primary Key for a character is not his name; let's pretend there is a unique ID attached to each character. When a character dies, just add this ID to the corpse information. This would allow a character to be referenced from its corpse, regardless of online status.
--

I just don't understand how it could be 'too hard' for faction to be learned from a corpse. You don't need to put the faction on the corpse, you just need to get the character record for whom the corpse belongs to, and extract faction information.

There _has_ to be a valid reason why you can't, or won't, do this. I'd just like to know what the reasons are.
