1. Droped frames 'flashing'

2. One pixel lines around in game items like Fire and Trees, I see the lines at the edge of the planes or ploygons of these items, it apppers the alpha maps are off.

3. Melting Interface, this only happens when I set the Hardware Antialiasing to 2x2 (special) and Force AA is checked. The interface menu items (not game window) 'melt' or 'smear' 45 degrees to the upper left. Other settings such as 4x4 or 2x2 work fine, and this makes things look oh so nice!

anyhoo, if I can figure out how to fix these I will post it, but I am still tinkering with settings...

For Nvidia GeForce and Quadro cards under 98SE Nvidia Ref Drivers W9X-650 12/14/2000 try this...

***
Note: This works for me, so it may not work for you, but please respond and let me know, thx.
***

Under display Properties choose Advanced

Then choose Quadro or GeForce Tab (the one with the spining logo)

Choose Additional Properties

Choose Direct 3D Settings Tab

In this window check the following:
Enable fog table emulation

Adjust Z-buffer depth to rendering depth if unequal

Enable alternate depth buffering technique

(uncheck) Display logo when running Direct3D applications

Set Mipmap Detail Level to 'BLEND'

Next choose More Direct3D

Set Texel Alighnment to '3'

Set Antialiasing to 2x2

check Force antialiasing in all applications

click ok and apply and give E.Q. a try (you may have to reboot)

These settings have cut down on the dropped frames on my system significantly but still have one pixel lines around in game items like Fire and Trees and Melting Interface, this only happens when I set the Hardware Antialiasing to 2x2 (special) and Force AA is checked.

Potential Bug with the new Druid/Shaman Healing spells: they are healing only 90% of what was listed on the test server.

Chloroblast:
Test - 428 HP / 175 mana (2.45 HP/mana)
Actual - 389 HP / 175 mana (2.22 HP/mana)

Nature's Touch:
Test - 978 HP / 400 mana (2.45 HP/mana)
Actual - 885 HP / 400 mana (2.22 HP/mana)

My theory is that the 10% server-side reduction of Cleric heal spells when cast by Shaman and Druids (e.g., Superior Heal, Greater Heal) is being applied to these two spells. That's fine since the casting time and the mana cost are the same...druids and shaman just heal a little less.

But these two new spells aren't cleric spells, and shouldn't be subject to this 10% reduction. This is especially true since these spells, when compared to similar cleric spells, already have some inefficiencies built in (longer casting time, more mana). There's no need to add the 10% HP reduction in as well.

E.g.,
Divine Light: 350 mana/4.5 secs
Nature's Touch: 400 mana/5.5 secs

Bottom line: the bug is the 10% reduction of the healing effect being carried over.
