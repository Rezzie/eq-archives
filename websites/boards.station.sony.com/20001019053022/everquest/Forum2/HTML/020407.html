Abashi
Station Member 		posted 10-11-2000 09:17 PM           
There are two issues. One is textures. When EQ was released the system requirements were a 4MB video card (which has 2MB of texture memory). Any time there are more than 2MB of textures that need to be viewable, the card (assuming that it is a voodoo1) swaps textures in and out to disk, causing a large drop in framerate. Increasing textures to the point where it is likely that that occurs causes decreased performance for those closer to the minimum specs.

The second issue is models. Global model definitions are kept in a global file, or files now. These files are quite large, often being something that we don't want to modify and require download of too often. Optional downloads aren't an option in this case because we can't have two separate versions of the same file pass as "valid".

-Gordon
