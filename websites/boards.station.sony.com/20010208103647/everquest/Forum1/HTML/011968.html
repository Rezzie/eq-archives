I have some solutions, i've done some research on DirectX and other matters, and this may help alot of people.

Though i've never had any trouble with Directx 8.0, because if you know how to solve its sound issues, you are fine. =) Now that dx8.0a is out, you should download that instead of 8.0. You can grab 8.0a at:

updates.zdnet.com/products/pr_1010.htm

Now, to fix 99% of your potential sound/crash/zoning problems with Directx 8.0+, you have to do a few things. =)

You see directx 8.0+ has a fault, it puts your soundcard into debug mode. Which can cause massive framerate issues, and poor quality to glitchy to no sound, even crashes. You need to download the DirectX control panel, which will allow you to disable all these useless debug options that are causing millions of people problems. Check it out at:

www.win98central.com/data...ctxcpl.zip

Download this and unzip it into your windows/system directory, and an icon will magically appear in your control panel for Direct X. Click on it, and check ALL tabs, and most likely, one or more of them are set to debug. Move ALL the sliders back to the left, and this will disable all of these machine problem causing options.

Live on happily with Directx 8.0+(preferable 8.0a) after this, and enjoy greatly enhanced performance. =)

Now for some other cleanup issues.

Second, goto Start:Run and type dxdiag, goto the sound card tab, and set the 3d acceleration 1 notch from the left. Exit dxdiag. (only do this if crashing to desktop)

Goto Start:Run then type: msconfig

Goto the startup tab, and UNCLICK everything there, and reboot your system.

Once your system is rebooted, hit CTRL-ALT-DEL, and you should see only one thing running in the background and that is "Explorer". Now, at this point, you have a nearly
completely clean system running, with your sound card turned very low, and the chance of error should be minimal.

Run the game in this configuration, and tell me how it goes. Performance freaks should try this, and note your framerate improves a whole bunch!

You have all you need to install it.

The instructions on how to install it are in the readme file.

The file titled "directx" is actually "directx.cpl"

You are not seeing the .cpl (the extension) part of the filename because Windows Explorer (by default) is setup to hide the extensions of known file types.
Here's the quick 'n dirty installation instructions:

For Win9x:

Copy the directx.cpl file to your c:\windows\system directory.

For Win2k:

Copy the directx.cpl file to your c:\winnt\system32 directory.

Then, open up Control Panel, and you'll see the DirectX icon there.

Some FYI for anyone running Windows 2000...

WINIPCFG doesn't exist in Windows 2000.

The equivalent tool is a command-line executable called IPCONFIG. There is no gui with it, so you have to run it from a Command Prompt.

Start -> Run -> Type: CMD -> Click OK.

In the Command Prompt window, type: IPCONFIG /? and you will get a quick summary of the command's built-in Help.

Example:

IPCONFIG /renew

That will "renew" all network devices in the system.

For what it's worth, the IPCONFIG command is also available with Windows ME (as well as WINIPCFG,) but the command line options are a little different.
