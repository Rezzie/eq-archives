There's a simple solution to the summoned items problem that Verant has never commented on. I'll give it for the fifth time:

1. Whenever a player disconnects (LD or camp, doesn't matter), the server should add an entry to an in-memory table recording the player name and the time. 60k of server memory would be enough to track the last 1000 disconnects, easily. If the table is full, throw out the oldest entry to make room.

2. Do not destroy summoned items on disconnect.

3. On login, look the player up in the table. If not found, or if found but the time is more than, say, 20 minutes, destroy all summoned items the player has. Otherwise, leave the summoned items alone.

There. No client changes required, so no patch to download. No data stored in the player database, so no pwipe required. Besides solving the LD problem, it also allows players to camp for short periods without losing their items (e.g., bathroom break, pay the pizza guy break, talk wife into not moving out because you play too much EQ break, whatever). Trivial server resource usage. Easy to implement, unless the server code is absolutely unmaintainable spaghetti.

If you like this idea, start new threads proposing it. There may be something I'm missing that would make it impossible to do (e.g., summoned items could have a completely different format on the server, making it impossible to keep them across disconnect), but let's at least get Verant to acknowledge the idea and TELL us that it won't work if it won't.
