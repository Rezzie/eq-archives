I believe what sunli was trying to describe was a dynamically assigned IP address.

Yes, you can tie an IP address to a message board posting if you have the IP address that the message was posted from. Even if the IP address was dynamically assign, you generally will see a trend in the user using the same set of IP blocks.

If a message originated from an IP in the same or near same IP address block as the IP that the user plays everquest from, you can conclude that the author of the post is the same as the everquest player, assuming the author's posting name is the same as one of their characters.

Basically, Verant would have had to contact someone from the message board to divulge the IP address of the poster if those IP addresses were not publically available. Once they received the IP address, they would compare it to their own logs of that account and make the conclusion.

I am, in no way, opposing or supporting any side to this issue. I am merely stating that it is possible to prove, with reasonable certainty, that a message board poster can be confirmed to be a particular player given the IP addresses and usernames used by that player.

Sunli, you are comparing apples and oranges. I can't track you because I do not know what IP address you are posting from.

When a person plays everquest, their IP address is logged. Different ISPs use different address pools. I work at a small ISP and we have about 144 dialup lines and have that many IP addresses assigned to them.

If someone were to log into EverQuest from any of those addresses, they would be logged on Verant's side as having come from one of those IP addresses. If that same person posted on a message board with the same name as one of their characters, but using a different IP address from our address pool, you can infer that it was him because the IP address range is limited for us, being a small ISP.

Many larger ISPs regionalize their IP address pools to ease assignment so one internet user can use a very small address pool every time they connect to their ISP.

Some providers have a huge address pool (AOL for example), and it is difficult to say for certain that 2 different examples of usage from the same large address pool are the same person.
