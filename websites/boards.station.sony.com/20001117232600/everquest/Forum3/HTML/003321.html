I had been unable to log in my main char after running TestEq Friday night and tried re-installing DirectX 7.0, reinstalling my Sound and Video Drivers, Thorough File Check on all 3 patches all to no avail. I could log in on my monk so I petitioned and was actually responded to within 15 minutes. This was my first petition, so needless to say after reading the horror stories on here I was amazed already at my luck. After talkin to GM Aetheria for a few minutes and explaining the situation I was told to try to zone with my monk who was in FoB because he/she (not sure) thought that the zone where my main was located was prolly bugged. Sure enough upon zoning I got dumped to the desktop, just like each time I tried to log in my main. So I once again ran thorough file check on each patch (Standard, optional, and kunark)

NOTHING! not one problem detected. Now couldn't log in my monk or Shaman since BOTH are in Cabilis. I deleted all the cabeast.* files and then ran the patches again... Thorough file check on Standard and Optional: NOTHING, regular Kunark patch notices missing files and re-downloads them. Problem solved, back in with both chars!

Cheer GM Aetheria for fast response, courteous service and accurate troubleshooting. Boo to Thorough file check since it did nothing for me...

Dunno if this is something that Thorough file check was designed to handle, but it certainly seemed like it should have caught these corrupt files. Might wanna look into it.
