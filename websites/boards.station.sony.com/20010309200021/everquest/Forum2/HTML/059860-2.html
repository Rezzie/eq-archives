Nazershaer said: "Another thing, a quick system wide warning would have been nice before the servers were taken down, it sure would have helped what will be a MASSIVE amount of res request from GMs."

No, a QUICK message is not sufficient. It has to be something that gives everyone a reasonable chance to see it independent of when or how frequently you log in. A system message will need to be posted for every log in for well over a week to reliably inform everyone. A week may not be enough. Therefore, that is not a good system because it will cause System Message Complacency.

The issue here is how to communicate an important change like this.

1. Changing the items stats: not good because one my never see it.

2. System message: Not good because although quite visible, you don't know how often a player logs in so you have to leave the message up for a long time. Long time messages cause them to be ignored. So this is not good.

3. Post on boards: Not good, only a few people read boards.

4. Post on web site: Not good, only a few people read boards.

5. Send an email to all users with change: Very good solution. Email is reliable. Verant has the email addresses of all users. People read their email frequently.

Why care? Because the result of failed communication results in lost of items for a user. Concievably, very hard to obtain items.

Actually the best is option 6: all of the above.
