In IE 5.0 (or greater) try this:

Click Tools
Click Internet Options...
Click the Advanced tab
Ensure that "Disable Script Debugging" is checked

Good luck. It worked for me.

Upgrade to 5.5, and the line 253 problem goes away (because there IS no error; If you look at the source, line 253 IS '}').

The 'document.REPLIER.UserName' one on replies is still there. That's because there is some javascript that's supposed to get your username from a cookie, or set the focus to the username field if it can't get the cookie. THERE'S NO USERNAME FIELD IN THE REPLY FORM. TELL THE WEBMASTER TO GET RID OF THE JAVASCRIPT STARTING ON LINE 77.

Note: You can see this in Netscape, along with some other errors that IE must be letting past, or are not errors in IE. Start a reply, then type
'javascript:' in the location and hit enter.
