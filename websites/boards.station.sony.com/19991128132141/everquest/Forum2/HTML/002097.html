Traith
Station Member 		posted 10-22-99 10:39 PM           
Right-click on <My Computer>, click on 'Properties', click on [Performance] tab then on the {Virtual Memory} button.

When the Minimum and Maximum numbers are the same, the swap file is 'fixed' as in, it's size will remain constant. Since EQ needs 200mb of virtual memory when running on a 32mb PC, 200mb should suffice. Heck set it to 256mb. That's enough for practically anything out there. When Windows complains (and when does it not) that you have insufficient resources, it's referring to 'System Resources' not this.

Put the swap file on the first partition of the fastest hard drive you have, with it's own dedicated IDE channel (ie no other drives shares the IDE cable). Ideally, it should also be the very first piece of data on that partition. So basically, forget all this if you have a PC with just a C: Drive. For you, do the below at least.

The swap file should be permanent or fixed and contigous as well. To get this done, disable virtual memory, reboot, defrag the partition you intend to put the swap file on. Then reenable VM with Maximum/Minimum settings being the same.
