The timer is reset each time the zone is brought down, Phalia. So basically the bucket is emptied each time, and then slowly fills again.

Hence, any time spent before a patch is annuled once the patch starts, in effectivity; timer starts over again once the zone is brought back online.

This also includes other effects that cause the zone to go down, like a server wide failure or a zone reset. These will all reset the spawntimers.

Frankly, if you're going to camp Rage, see if he even can spawn in the time between your current time and the next scheduled patch. The timeframe in which he can actually spawn is set and not too wide, really. (0-2 days after zone is brought back online, 4.5-7 days after previous kill).

Mmm, I believe we are mixing up two things here.

1) Spawn timer: this is purely based on the game engine, on when it is supposed to generate what mobs in the zone. For Ragefire, this timer is around 0-2 days after a patch, and 4.5-7 days after previous kill. This is the technical game mechanic behind his spawn. Each time the zone is brought down, this timer is reset.

2) Camp timer: this is purely an inter-player defined timer. Has to do with courtesy between players; someone can do 500+ hours on Ragefire and other may value this effort by letting this person get the spawn when it finally spawns while he is online. This is not related to the mechanics of the game, but purely inter-player courtesy. As courtesy, time spent at the camp - even between patches - is usually added up on that players account.

Spawn timers and Camp timers are NOT the same, by definition.

I was talking about the spawn timer. I think Phalia was talking about the camp timer.
