I know several people that ran into this problem with their custom UIs. I use Ortin and ran into it.

Basically, when trying to load the UI, I'd get an error and it'd dump me back to default. Looking in the UIErrors.txt file in the everquest directory would show this error:

[Fri Aug 29 13:34:15 2003] [Line:77 Source:UIFiles\Default\EQUI_Templates.xml] Couldn't find class:item 'A_QMarkBtnPressedFlyby' reference in FieldParseItemOfClass()

[Line:199 Source:UIFiles\Default\EQUI_Templates.xml] Couldn't find class:item 'A_QMarkBtnPressedFlyby' reference in FieldParseItemOfClass()

[Line:321 Source:UIFiles\Default\EQUI_Templates.xml] Couldn't find class:item 'A_QMarkBtnPressedFlyby' reference in FieldParseItemOfClass()

[Line:443 Source:UIFiles\Default\EQUI_Templates.xml] Couldn't find class:item 'A_QMarkBtnPressedFlyby' reference in FieldParseItemOfClass()

[Line:566 Source:UIFiles\Default\EQUI_Templates.xml] Couldn't find class:item 'A_QMarkBtnPressedFlyby' reference in FieldParseItemOfClass()
[Fri Aug 29 13:34:15 2003] Error loading skin: loading default skin instead.



The fix for this is to edit the EQUI_Animations file in your custom UI. You need to add the stuff listed in this post: http://boards.station.sony.com/ubb/everquest/Forum3/HTML/046083.html

Easiest way to do it is to edit the file, go to the very bottom, and just above the </xml> tag, cut and paste in the quoted section from TSR-Roger's post at that address.

Hope this helps you out! Drove me nuts tonight...

EQInterface.com always lists the XML changes when a patch comes out. If the changes show up on the Test server first, then they usually have the changes out days ahead of time.

First thing I always do after I've downloaded a patch with UI changes, is cancel the patch program, hit that website, and see if the changes affect my UI. Luckily, the author of my UI did a damn good job of making his "patch proof." :D
