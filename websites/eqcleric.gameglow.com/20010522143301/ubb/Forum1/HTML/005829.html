Talura, that is already similar to how the data is sent to the client now. There is no message from the server that actually says "RangerX is hit for 200 damage." It is sent from the server as a numerical sequence. The client just interprets the sequence and displays the message, but it primarily uses the data in the sequence to change RangerX's hitpoints. All that happens when you turn off "others hits" the client skips displaying the message, but the client still recieves the data from the server for reduction in RangerX's hitpoints.

So what they are going to do is only send hitpoint data every 3 or 6 seconds. Thus, there will be a dramatic jump in hitpoints on raids. This worries me also, this has the potential to really mess up complete heal timings and think how hard catching that enchanter is going to be.

But I also believe something needs to be done for raids. I guess we all have a love/hate situation here.

The issue now is that with client-side filters, you get the info whether or not you use it. If you filter out graphics, the data comes anyway, it just isn't displayed. If you filter out damage shield messages, guess what? - the data is sent anyway you just don't display it. With server side filters, the server won't send the data in the first place.

For people who have problems going LD during a raid due to poor connections (read more data arriving than their ISP link can handle), this will be a tremendous improvement. For those of us lucky enough to rarely go LD, things will remain exactly as they are since we won't be filtering.
