Zombie 		posted 04-13-2001 08:23 PM         
Well the first question would be, do you have libEQ.a in /usr/lib or /usr/local/lib? If not, redo step 3: Save the file http://libeq.trifocus.net/libeq/i386/linux/libEQ.a as /usr/lib/libEQ.a (again, using Netscape). If trifocus is down download http://www.krisp.com/libEQ.a.gz as /usr/lib/libEQ.a.gz and do gunzip /usr/lib/libEQ.a.gz.

If it IS in there but configure still cannot find it, do make distclean and configure again, if that doesn't work delete the entire linux directory (rm -R -f linux/*; rmdir linux) and start over at step 17.

adenine!
Zombie 		posted 04-14-2001 11:28 PM         
It will work, you just need the all of the needed libraries. Generally you can tell if you don't have some needed library installed, and which library that is, by the error message that configure gives you. Basically make sure you have libpcap-0.4-16, some version of gcc, gcc-c++, and libstdc++ (Any version should work but I know for a fact that 2.96-54 works since that comes with RedHat), and autoconf, automake, and libtool. That's probably all but I don't know for sure.
