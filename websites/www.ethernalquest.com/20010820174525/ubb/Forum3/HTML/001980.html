Newbie's guide to installing ShowEQ CVS in FreeBSD 4.3

I wrote this guide because I want to help all my fellow FreeBSD users (we all know its the bext UNIX variant out there). I will try to update it everytime a major bug release is posted to CVS.

First off, I should let you know that ShowEQ will be REALLY slow updating it's display. In linux your map will move nice and smooth, and all the on screen displays will update in real time. In FreeBSD, this usually takes about 20-30 seconds. It's a VERY noticeable jump. Maybe one day the developers will get around to fixing this bug, *shrug*.. anyway, don't bug them about it. They don't get paid to put up with your crap, you have been warned.

You must be logged in as root to do this. Remember that anything that is not a plain English sentence is a command run in the console! E.g. the text after the number should be typed in the console!

1. Install the following ports:

x11-toolkits/qt23
graphics/gd
graphics/png
devel/libgnugetopt
devel/autoconf
devel/automake
devel/libtool

2. Download my custom CVS mirror/hack from http://irev.net/~bubbles/showeq.3.0.2.r1-freebsd.tar.gz
2a. Save the file http://libeq.trifocus.net/libeq/i386/linux/libEQ.a as /usr/lib/libEQ.a (with Netscape).
If trifocus is down download http://www.krisp.com/libEQ.a.gz as /usr/lib/libEQ.a.gz and do gunzip /usr/lib/libEQ.a.gz
3. tar xvfz showeq.3.0.2.r1-freebsd.tar.gz
4. cd linux
5. make -f Makefile.dist
6. env LIBQT=-lqt2 MOC=/usr/X11R6/bin/moc2 ./configure --with-extra-includes=/usr/local/include --with-extra-libs=/usr/local/lib --with-qt-includes=/usr/X11R6/include/qt2 --with-qt-libraries=/usr/X11R6/lib; gmake
7. cd src
8. Open Makefile in your favorite text editor, scroll all the way to the bottom and uncomment the following section (remove the # from the beginning of each of the following lines in the file):

#install-exec-local: $(noinst_PROGRAMS)
# $(mkinstalldirs) $(CGIDIR)
# @list='$(noinst_PROGRAMS)'; for p in $$list; do \
# if test -f $$p; then \
# echo " $(LIBTOOL) --mode=install $(INSTALL_PROGRAM) $$p $(CGIDIR)/`echo $$p|sed 's/$(EXEEXT)$$//'|sed '$(transform)
# $(LIBTOOL) --mode=install $(INSTALL_PROGRAM) $$p $(CGIDIR)/`echo $$p|sed 's/$(EXEEXT)$$//'|sed '$(transform)'|sed '
# else :; fi; \
# done

9. cd ..
10. make install
11. cd maps
12. cp * /usr/local/share/showeq/
13. cd ..
14. cd conf

ONLY PERFORM THE FOLLOWING STEPS IF THIS IS YOUR FIRST TIME INSTALLING SHOWEQ ON YOUR MACHINE. IF YOU DO THESE, ANY PREVIOUS SETTINGS WILL BE LOST.

15. Open showeq.conf.dist in your favorite text editor, at the very top, be sure to change "Device" to the identifier for your network device (example: dc0). Be sure to change "IP" to the IP address of the machine you wish to sniff packets (the one you play EQ on).
16. cp showeq.conf.dist /usr/local/share/showeq/showeq.conf
17. Open filters.conf.dist in your favorite text editor, make any changes you want.
18. cp filters.conf.dist /usr/local/share/showeq/filters.conf


Congratulations, everything should now work. Sorry if I pissed anyone off by making a custom tarball, its easier this way.
