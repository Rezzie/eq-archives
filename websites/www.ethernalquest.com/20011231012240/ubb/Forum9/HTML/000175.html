This one works - thanks to a guildmember for testing it . Has been tested on WinXP and Win98SE.

-------------------------------------------

This tool creates a window that sits in the bottom right corner of the screen, and display all relevant details on the missions given by a mission terminal (location, cash, and item reward), without having to manually shift-clicking each mission and scrolling to the bottom of the description to find out what you're getting.

http://muzzleflash.da.ru

-------------------------------------------

Things planned for future versions:
1/ Showing the 'mission retrieval' item in addition to the reward item.
2/ Obtaining item descriptions directly from AO, so the tool doesn't need to maintain it's own database of items.
3/ Ability to set an item watch (eg. for 'Battle Suit') and automatically selecting the request mission button until it finds one.

Looking for help in the following areas:

1/ Details on how to access item and playfield descriptions from AO directly.
2/ Someone who tested it on Win98SE stated that the window wasn't staying on top (over the AO window). It works on WinXP. I'm just using the SetWindowPos API call to set AOMD as the topmost window - any ideas on why 98 doesn't comply?
3/ People have mentioned that packets can also be intercepted via Winsock, allowing the tool to work with dialup modems. I'd appreciate any info on how to do this

Decaying Skeleton 		posted 12-17-2001 01:58 AM               
The first thing to determine is if it is an AOMD or WinPcap problem. To do this, please try 'Analyzer', a free WinPcap packet capture & view utility by the people who made WinPcap. It is available at http://netgroup-serv.polito.it/analyzer/ . If that utility doesn't work either, then it is a WinPcap problem, and you might like to check the WinPcap FAQ at http://netgroup-serv.polito.it/winpcap/misc/faq.htm to determine why it isn't working with your network card. One possible reason is you are running WinXP and didn't install the latest WinPcap v2.3beta, the only one that supports WinXP.

If Analyzer *does* work and captures/displays packets correctly, then please let me know, as it would be an AOMD problem.
