Zombie 		posted 04-13-2001 07:56 AM         
Well how does the router work? Does it take the real internet IP for itself and give out a 192.168.x.x or other nonroutable IP to each computer behind it? If that's what it does, I don't think you'd be able to just plug the EQ and Linux boxes into the router and make ShowEQ work, because the router would send the packets straight to the EQ computer without going to ShowEQ, unless there is some way you can configure your router to broadcast all packets.

If that doesn't work you'll need 2 NICS in the Linux box, with one going to the router and one going to the EQ machine, and have IP Masquerade running on Linux (follow WhiteFire's guide or read a Masquerade HOWTO on http://www.linuxdoc.org). I can't say for sure what you need to do since I don't know exactly about your router and how it can be configured.

Zombie 		posted 04-13-2001 09:26 PM         
You would run it on the Linux box so your network would look like this:
Router----Other computers & cable
|
|
Linux box running IP Masq
|
|
Windows box set to use Linux IP as gateway

The term "IP Masquerade" is just an idea, you will use one of three programs, ipfwadm, ipchains, or iptables depending on which kernel you have (probably ipchains) to set it up. WhiteFire's networking guide on this board describes the general process using ipchains. For a more detailed description and setup go to linuxdoc.org, click HOWTOs and read the IP Masquerade-HOWTO, however, that may be a bit intimidating at first, so Whitefire's guide is probably the best for now.
