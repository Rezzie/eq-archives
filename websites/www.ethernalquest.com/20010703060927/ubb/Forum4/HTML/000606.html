its not that bad ... put your modem on your linux box (preferably a exteral as many internals are wins modems)

configure it with RH ppp dialer or some other program.

if you can get your modem working in linux and connected to the internet the rest is as follows

set your linux box and windows box up on the same internal private net....ie both machines network cards have a ip address of something like 192.168.0.n and 192.168.0.n+1

then set your gateway on your windows box to the ip address on your linux box

set DNS to what ever your ISP's DNS is on your windows box.

make a file called rc.firewall in the /etc/rc.d directory with the following lines

********* rc.firewall contents below ********
#!/bin/sh

echo 1 > /proc/sys/net/ipv4/ip_forward

/sbin/ipchains -A forward -s 192.168.0.0/24 -d 0.0.0.0/0 -j MASQ
********* rc.firewall contents above ********

obviously don't include the lines with the *'s
this sets up ipchains to let any machine on your internal network with a ip address of 192.168.0.x to get onto the internet
the echo line is something to do with ipchains...if you don't put that line in then ipchains will tell you to type it before running the last line.

Every time I reboot I just go into my /etc/rc.d directory and run
./rc.firewall
then connect to the internet
now since all traffic goes through your linux box showeq can sniff the packets

to be more security minded you can set up more of a firewall but EQ needs special configuration done on the firewall to work as the servers need to be able to get throught the firewall...but thats a whole bag i'm still trying to get working
