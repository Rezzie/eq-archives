When using PPPoE the DSL modem does _not_ talk IP to your windows box. It talks its own protocol that is translated to IP at the driver level, or, at least that's how it is with me.

thus if you sniff everything on your LAN and don't filter for IP, you should see a bunch of data flowing when the PPPoE connection is active.

hope this helps somewhat.

I have this configuration, and to run ShowEQ I had to set up a box to do gatewaying, which was a pain due to my provider, but it was the only way

Decaying Skeleton 		posted 06-08-2000 04:08 PM         
You can use a broadband router, like the Linksys Cable/DSL Router, to handle all the gateway/IP masquerading tasks.

If you take that option you will need to use an addition router or switch in order to get ShowEQ working. Linksys just came out with a new version of their router which has a single uplink port and no built-in switch, for about $110, $70 less than their original 4 port router/switch combination box.

My current working setup is: DSL Modem -> Linksys DSL Router -> generic 100Mb 8-port switch -> {my computers}

Note: ShowEQ won't work without the additional switch, even if you have the 4-port Linksys box, because the switch in the Linksys box won't send packets out on a port that other than the one that it knows the destination machine is connected to.

I have the same setup as you, my DSL uses PPPoE also, What I had to do is install a PPPoE dialer on the linux box, and setup IPMasqing. To make the linux box a sort of proxie. The reason why this can not be done any other way, is because PPPoE uses an Encapsulated Packet structure, and showeq can not sniff the packets, until they have been converted to Standard IP.

The software you need is located at: www.roaringpenguin.com
