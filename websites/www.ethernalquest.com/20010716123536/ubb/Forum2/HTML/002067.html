I haven't heard anything back from Ashran on this, but I noticed a misalignment between the itemStruct that SEQ is using and that ItemDB.exe is using. ItemDB is the utility that imports the .crp files from the item database script and (optionally) converts them into .db files that EMU uses. The misalignment starts at 0xA8 (168 dec) in the SEQ itemStruct where STR is, the ItemDB.exe has STR starting at 0xA4 (164). From here on, the entire structure is shifted by 4 bytes until the last field (the "placeholder" field which is 26 bytes long in SEQ and 30 bytes long in ItemDB.exe).

Is this a bug in ItemDB.exe or is EMU using an older format of the item struct (like maybe VI changed the structure for some gay reason and EMU is still using an older struct because it uses an older client?)
