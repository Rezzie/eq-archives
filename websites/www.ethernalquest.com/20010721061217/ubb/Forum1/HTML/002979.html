Asking what language is best is kind of a "religious" issue.

However, it's generally accepted that Basic, as a higher level language, is more productive "per-line" than C/C++. C and C++ are what we call "low-level languages" because it's essential that the programmer rather innately understand the memory layout of their data and because it's made to give the programmer's close to complete hardware-level access. (reference Code Complete - 1993 and at least 3 HUGE corporate studies)

Where, in VB, you might have something like the Replace() function as a single keyword, in C / C++ it would take up at least a couple of lines. That's how productivity works, as a metric. It's just a raw comparison between the number of commands needed in one language to replicate the functionality of the equivilant in another language.

Since VB is a higher-level language, and since it runs more slowly it is generally taken less seriously as a "hard-core" language.

I mention this because the choice of language will depend on your platform. Since you said that you used VB for the first version, that implies that you're stuck in Windows. That's not bad, but it means that you'll need something that generates Windows executables. That means that you'll need something like the Microsoft packages (you've tried their VB), try Visual C++ (Visual Studio). You could also go with Borland's C++ Builder or Borland's Delphi (not C/C++ based, but object oriented Pascal. Pascal would be more language productive but you'll pay for it in the short term because you'll have to learn this new language. It compiles to the speed of C/C++ (it has the same back-end compiler as Borland's C/C++).

Borland and Microsoft BOTH put those products in great environments.

If you CAN go Unix, there are many.. but GCC is, I believe, the defacto standard.

Ok, some people here aren't computer scientists. I've been programming, have had articles published, have lectured and have written commercial games over the last 20 years -- that doesn't make me right in every circumstance... but I'm explaining where I'm coming from.

The definition of a high level langauge is :
how much it abstracts the proximity of the underlying architecture (kind of, how much more natural the language is, so a more English-like computer language is considered higher-level than C/C++ precisely because C/C++ was designed to get "down" and give you low-level access).

Whether the language can support operator overloading, or objects is irrelevant. There is something called Object Assembly, and I've even seen something along the lines of an Object Cobol. That doesn't change their "level".

"I don't flame people much... i rarely do but i gotta now. C++ is a high level language, VB is fucking paint by number programming ok? when you say this and that C++ high level language more powerful then VB, if you don't know anything about C++ don't talk about it ok? "

I never mentioned the power. I was simply pointing out the facts. C and C++ are designed for programmers. Basic and Pascal were designed for more readability and ease of understanding.

Technically, C++ is a mid level language, as implemented. But Basic and Pascal are still higher level (Ada is about the highest).

Now, you probably still won't believe me, but here are the facts from HUGE studies :
"data shows that programmers working with high-level languages achieve better productivity and quality than those working with lower-level languages. Pascal and Ada have been credited with improving productivity, reliability, simplicity and comprehensibility by a factor of 5 over low-level languages." (source was Brooks, 1987, No Silver Bullets -- Essence and Accidents of Software Engineering)

The table compares each language to the average number of statements required to perform a given function (with machine-language / assembler as the base)

Assembler 1 to 1
Ada 1 to 4.5
Basic 1 to 4.72
VB 1 to 4.8 (not a big savings)
C 1 to 2.5 (C as designed)
C++ 1 to 2.9 (C++ as used)
Fortran 1 to 3
Pascal 1 to 3.5
Delphi 1 to 3.8
( Source was Applied Software Measurement, Jones, Caper 1991 with supplements released 1999).

What I believe that you mean is that C++ is more complicated and considered a more serious language. For the most part, I rather agree. Nothing against Visual Basic... but VB is simple because Microsoft made some -- tradeoffs. You get that level of simplicity by giving up the ability to do some things easily. Very VERY often in VB, you will run into a seemingly simple task that VB apparently makes impossible or absurdly difficult. It's as though you have to jump through hoops to do something.

A language that has been designed so that certain tasks become difficult is poorly designed (my opinion). With most sensible languages, like C and Ada, (pretty much) anything you want to do is doable. It takes time, effort and code... but it's not usually like banging your head against the wall.

But the intent behind the design of a language actually means nothing. Pascal was also designed to be a learning language. It has evolved to encompass objects. VB is also profoundly different from Basic (as it was originally conceived). VB is, from what I think most people would agree, no longer Basic. There are no line-numbers, it has named routines (functions and the like), it supports abstract data-types, the likes of which were beyond the pale of original Basic. You can have local and global variables.

So, the fact that Pascal and Basic were designed for teaching actually means nothing nowadays. The fact that C is a low-mid level language means nothing in "production" since C is more accepted.

All the "level" of a language means is that people using it will be more or less productive.

A good example is... my first language was assembler language (for the 6502). I'm very good with assembler (aka assembly). I can recite the numerical opcodes for different addressing modes of instructions for several generations of CPU. It's my lingua franca. But producing a quicksort, or binary search, or word-graph parsing routine would still take me MUCH longer in assembler than it would in C++ or Basic.

Why? Is it because I suck at assembler? No.. It's the level of the language. Everything is faster than assembler, programming wise (it has to be, because nothing can compile to LESS than one instruction).

Dys, he's not comparing Pascal to C++ in any real way.. I've had some experience with both Pascal and C/C++.. And I've found that Pascal can be very easily translated into C, but Pascal is limited in many ways that C is not.

Pascal as stated, is a total learners language. It's fuctional, and while it may have some practical uses, it is not a fully functional language. I have no real experience with Delphi, I have heard that it does an ok job of combining the power of Pascal with some of the ease of use of VB, but I have neither used VB nor Delphi so I can't really comment on this analogy.

Btw.. To the person who referred to HTML as a language, it is not. Java is also not a combination of C and Java, this is just a misconception people have because Java is a popular language for webdev.

HTML is, substantially, a text-format. It's like doing what your word processor does except by hand. There are no variables, no loops, no pointers, no strings, no nothing -- just text and 'tags' which modify the text.

JAVA is, substantially, a multi-platform version of C++. This is maybe an oversimplification of it, but it is in essence the truth, it basically sacrifices power for compatibility rather than what most languages do, power for simplicity (although it is also a bit simpler than C++ as a result).

This is why JAVA is so popular for webdev, it and the hybrid 'Java-Script' make doing the impossible in HTML, possible, and allows people to share code on multiple platforms and performs totally adequately for MANY web applications (major databases and forums are of course an exception, that's where, as you mentioned, PERL/PHP/etc come in).

Perhaps you were thinking of Java-Script when you called java a cross between C and HTML, because javascript is this basically, javascript is just a neutered version of JAVA that can do loops and a few other things HTML can't do (because as I mentioned, HTML isn't a language).
