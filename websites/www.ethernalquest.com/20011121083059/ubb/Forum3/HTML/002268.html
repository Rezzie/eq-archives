Decaying Skeleton 		posted 10-02-2001 10:40 PM         
It sounds like a lot of you are relying on your hubs to "broadcast" the EQ info across your networks so that ShowEQ can pick up the packets. This, while in theory, works and will continue to work, it's confusing to some of you who've purchased switches (which by nature do not broadcast). I would recommend setting up your network like so...

First using your DSL router (I assume you have the Cisco...right?) set up NAT to forward all packets to one IP. The command at the Cisco CBOS is "set nat entry add 10.0.0.2" (assuming your linux box is 10.0.0.2). This will forward all data straight to the network card on your linux box. You will need to use a crossover cable to connect the Cisco to the Linux box. From here you want to install a second network card into your linux box (this makes your linux box a dual-homed machine). The second network card will use an ip like 192.168.0.1 and will be your "internal" network that the rest of your computers are on. This network card will plug directly into your hub or switch (this method doesn't care if you're using a switch). Now what you want to do is setup Masquerading (NAT) on the linux box so that packets will route between the two network cards...use the following commands to do this at a command prompt:

echo 1 > /proc/sys/net/ipv4/ip_forward
iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

the "eth0" refers to the network card that has the 10.0.0.2 address (run ifconfig to see which that is).

These particular instructions assume you are running a 2.4 kernel that has Netfilter/IPTables compiled into the kernel. If you don't know how to do that I will try to link you to a decent kernel upgrade/recompiling article if you reply.

In this way, Linux actually becomes your "router" which can allow you to do some very cool things as far as security goes, but will also allow you to use ShowEQ a lot more reliably (granted, it does require a little more effort).

Hope this helps some of you budding Linux/Network geeks out there...

Dave

Almost forgot...I run 3 showeq sessions without problems on my box. since all data destined for the internet goes through this box it doesn't "miss out" on anything.

Zombie 		posted 10-04-2001 09:14 AM            
wouldnt be a bug in showeq, your libpcap is more likely the cause of confusion. When the EQPacket class initializes itself it sets up a pcap filter to gett packets off your network, it sets the filter up to only return packets that match a certain ip address, so your showeq should only be seeing the packets for one of your boxes. Other boxes running EQ shouldnt confuse showeq, since showeq never sees those packets... unless your libpcap is broken and somehow is not really filtering properly by ip address.
