Yes you can invite people into the guild with this hack. But thats It. You can't change the MOTD only invite and un-invite. I haven't tried to see if it can make you guild leader I will leave that up to someone else. I like my guild too much.

start become a guild officer
4c1c64 80b84b10000001 c6054b10000002
4c1c6b 730f eb0f
end

when you invite someone into the guild it does a compare at address 4c1c64 checking bytes at [eax+0000104b] with hex 01 to see if you are a guild officer. If you are not then it jumps at 4c1c6b to msg "You are not a guild officer" By changing the statement to a MOV (Move) instead of a CMP (Compare). Your are able to move byte 02 into [eax+0000104b] making it jump to where you invite so & so into the guild. Just for added insurance I changed 4c1c6b 73 (jump statement JNB) to EB (jump Always). So, here you are! Use at your own risk

I use two programs for hacking. W32dsm89 and Softice. W32dsm89 is a disassembler\debuger, it allows you to disassemble the EQgame.exe file into assembly language. And Verant being ever so tight on their programers, they have the comment everything. Makes it much easier to find stuff. So after disassembling the program I did a text search for Guild officer and there it was I also use softice disassembler debuger that runs in memory so when in the game it's self I can do a CNTL "D" and enter the game memory and disassemble the game, in the game, to look for stuff. For instance I dont use the same SOW that is on this board I did a BPM (Break Point Memory) at memory address 67e335 then it listed all the code that made reference to that address. I found one that does a CMP(Compare) at that address to see if it's walk or run. Walk being 00, run being 01. So change the CMP to a MOV (Move) moving hex 02 into where the 00 or 01 is stored and I have SOW.
