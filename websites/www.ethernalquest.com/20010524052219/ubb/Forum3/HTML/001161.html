Which SCSI Card and Hard Disk do you have?

If its a regular old Adaptec series card you should be ok.. otherwise you'd have to check to see if that model of card is supported or not. Worst case if it can't be resolved and IDE is an option for you.. then by all means give it a shot.

One other thing to try.. make sure the drive is completely wiped clean and any existing partitions are removed. You can do this by using a FDISK or partitioning utility. A Win98 boot disk could do this.. just run FDISK and delete all partitions on the drive. Reboot and retry the Linux installation. Recreate the partitions from Linux setup. I'm only saying this cuz you never know.. a goofed up partition table or some other wacked thing with the drive may cause these problems (technical term). The safest route is always to start with a totally clean drive.

hate to ask the obvious but did you make sure that you installed lilo into the MBR (Master boot record, which should be default) of your primary drive?

Also if you are able to make a lilo boot disk you might just boot off that, make sure that your kernel is properly compiled for the system and SCSI card then you can actualy edit the lilo.conf file in the /etc directory to install to the MBR of /dev/sda then from the command line type lilo, reboot without the disk and see if it works.

for help with compiling the kernel check the linux sites or if you feel comfortable with just plunging in:

cd /usr/src/linux
make mrproper
make <config>|<menuconfig>|<xconfig>
make dep
make clean
make <zImage>|<bzImage>
make modules
make modules-install
make install (only works on some)

if make install doesnt work you have to go into the directory that it made zImage (or bzImage) and install it by hand, but I am too lazy to go that in depth.
