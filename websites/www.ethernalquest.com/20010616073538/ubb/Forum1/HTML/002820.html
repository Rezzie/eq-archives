Zombie 		posted 06-04-2001 11:02 PM            
I used mandrake and windows on the same system for a long time before i setup a seperate system for linux. I didn't use the LILO boot manager as my main bootloader though. If you want to run windows and linux on the same system here is what you do. Backup all your windows dookie, your garbage files, porn, what ever you wanna keep. If you made a recovery/uninstall disk for windows reboot with that disk after your done backing up. Run the FDISK command at the prompt. Destroy the partitions on your harddrive and then remake them. It really depends on how much space/memory you have. You wanna set your windows partition so that you leave at the very least 5gig or so for linux. So if you have a 10gig disk, go to create primary partition, then set it to 50% of the drive. Reboot and reload WinME, I think the install for ME will format the partition for you. Then when you get mandrake boot from the CD and setup. I use the expert install, because I like to choose the partitioning myself. If you use custom it will partition the disk for you. Plus, if you use expert you can get ride of some of the useless packages. The packages all have descriptions so you'll have no trouble figuring out which ones you don't want. I would set about a 500-700meg swap partition, and partition the rest as ext2 for root. I use to use 2 seperate harddrives, and i downloaded a freeware bootloader call Bootus for selecting the operating system that I wanted on startup. With bootus you can make a recovery disk that will reload the boot manager for you. That way when windows craps out (and it will) you can copy all your data into your linux partition (while in linux), reload windows, reload the boot manager, and copy all the data back over. If you get stuck your welcome to drop me an email.

"As long as your kernel and rpms are somewhat recent, thats all you really need to care about."


and thats why us debian users will never touch an rpm based distro. debian uses deb packages rather than rpm, and has a frontend tool to locate, download and install packages in one step. Debian's unstable tree is updated multiple times daily, and the command

apt-get update && apt-get dist-upgrade

upgrades your entire system to the latest versions of all packages. I've get to see any rpm based distro comptete in terms of up to date packages as debian, and the debian package maintainers do a great job ensuring that the system works well together and that they conform to things like the linux filesystem standard.

use apt, and you will never look back

as an example, the day libc 2.2.3 was released there was a debian pacakge, and my daily update routine put it on my system, that kind of updating just doesnt happen with rpm. i'm also opting to use gcc-3.0-prerelease and a beta version of qt-3.0.0 (rather than gcc 2.95.4 and qt 2.3.0). i like debian
