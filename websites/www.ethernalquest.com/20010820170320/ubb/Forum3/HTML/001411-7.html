Zombie 		posted 06-01-2001 04:29 AM            
You're trying to install a qt-devel rpm without installing its corresponding qt rpm (not going to work). The guide assumes you're using a clean copy of RedHat. Download the files ftp://ftp.redhat.com/pub/redhat/redhat-7.1-en/os/i386/RedHat/RPMS/qt-2.3.0-3.i386.rpm and ftp://ftp.redhat.com/pub/redhat/redhat-7.1-en/os/i386/RedHat/RPMS/qt-devel-2.3.0-3.i386.rpm to some directory, cd to that directory, and run the command rpm -Uvh qt-2.3.0-3.i386.rpm qt-devel-2.3.0-3.i386.rpm

Then start the guide again from step 8 and it should work.

Decaying Skeleton 		posted 06-26-2001 04:52 PM         
I think one of the problems with the guide is that some people that are installing redhat don't realize that when installing 7.1, you don't have to do the RPM steps.

redhat 7.1 has all the features IF you install them when loading up the system initially. If you specify a Custom install and then grab the libpcap, libstdc++, and qt packages, then you don't have to do the RPM steps. Or if you do a full install when you are doing a Custom install, then all the related packages are loaded.

If you did a full install of redhat 7.1 under the Custom install, then skip the RPM steps in this guide. BUT do all the others!
