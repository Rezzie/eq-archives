Zombie 		posted 05-01-2001 04:16 PM         
Ok... note to self ... *no one reads the CHANGES file*

Those of you having problems with the latest CVS version would have solved them immediately by reading the CHANGES file. If you are unable, or unwilling to install the latest QT (2.3.0) - you can still build the semi-latest SEQ (2.5.4) by checking out SEQ from before the patch...

See the CHANGES file for more information.

Also - please remember that the CVS is a development version of SEQ, which means it is constantly changing and/or may not compile on your machine. As such, I reccomend two things :

1. You always keep a copy of at least 1 working version of SEQ ... just rename the executable file to showeq.working or something and keep that in reserve.

2. Do not use the CVS version if you are not willing to assume the risks of using virtually untested code, or are unwilling or unable to do what is necessary to get it to work (such as reading the CHANGES file and/or installing QT 2.3.0)

QTDIR is already set when the computer starts up so there's no reason why it should need to be defined again unless QT was upgraded and QTDIR isn't pointing to the new directory, but on a default install it does not need to be set.
