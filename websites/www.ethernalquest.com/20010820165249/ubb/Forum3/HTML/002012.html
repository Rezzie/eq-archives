...well actually its runing under linux thats running under vmware that's running on win2k, but that wouldn't have made it such an interesting title. The way it works is you:

1) install vmware and follow their instructions to install a guest OS (Mandrake 8 in my case)and make sure u setup a bridged network (through VMWare).
2) Intall showeq on the linux 'os' per adenine!'s guide and bada-bing! Your in business.

I'll admit its a bit cumbersome at first but it removes the hardware requirement of using showeq.

ps- in order to have eq running from a window eqw is a must, lookup the link from elsewhere on this board. This setup is should be reserved for advanced users of windows and n00bs to linux or if u feel confortable with messing around with your networkin; bleh its realy not a big deal!

This option is great! I am a linux n00b, but I am an old hand at Windows (come to think of it, who isn't?), and these instructions were almost flawless. I installed RH 7.0 on VMWare, and then followed the instructions that Arch pointed to. At the end of it, ShowEQ loaded up the first time and started sniffing the packets fine. Quick note: I am running EQ on a seperate comp, so I am not sure how well this config works on the same comp. However, a question has arisen.

1. SEQ only appears to run under Xwindows (I have Gnome loaded). I tried launching it w/o loading Gnome, and I got some error about X server. Also, I can only lauch showeq when I am logged in as su. Any ideas on these two?

Also, a note to other n00bs configuring this setup. From what I have found, and I may be wrong, but you have to perform all of the install steps as root, or you will get a lot of permission errors.

And I think I have a solution for the users that are seeing SEQ be slow under this setup. First, to do this, you have to have a good chunk of RAM. My comp has 256mg, and under the configuration editor for memory in vmware, I alloted 160mg. I did that because SEQ was very slow and practically unusable even when I had it set to 128mg. I guess that 32mgs makes the difference. Also, under the global preferences menu, under the Memory tab, I set the total reserved memory to 204mgs.

Hopes this helps and thanks in advance for any insight anyone can offer to my questions. And thanks to everyone on this board. I could not have done this without this board (and the search feature)
