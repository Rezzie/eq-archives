Maps load properly, but I haven't figured out what packets say "You are in zone this and that" properly

Anyway, I thought I'd post the prebuild requirements for Odin's Eye, so people that want to be first can use it the minute it's released

Install RedHat 7.2
You're welcome to use anything else, but RH 7.2 works for me, and it's what this was built on.

Install a recent 2.4 kernel
The one shipping with RH 7.2 (or the updated one) will work. The important clue is to have iptables with full masquerading support.

Check OpenGL Functionality
Run "/usr/bin/gears -info". After 10 seconds or so it starts printing FPS information. If this is below 100, or if the GL_RENDER line doesn't include the magic word "DRI", then try to fix your OpenGL support.

Download and build qt-3.0.0
Remove the qt3 RPMs if you installed them
Download ftp://ftp.trolltech.com/qt/source/qt-x11-free-3.0.0.tar.gz

cd /usr/lib
tar xvvfz qt-x11-free-3.0.0.tar.gz
mv qt-x11* qt-3.0.0
cd qt-3.0.0
export QTDIR=$(pwd)
./configure -shared -thread
make

This will take F O R E V E R

Once done..

cd examples/opengl/gear
./gear

There should be some spinning gears on your screen.

Next, set up your DAOC machine to use your linux box as it's default gateway.

If they are on the same LAN, use the following on your linux box

for h in $(find /proc/sys/net/ipv4/conf -name send_redirects); do
echo 0 > $h
done

That will turn of automatic redirection so your daoc box will KEEP the linux box as it's default GW

Do some tcpdump without promiscous mode to see that you get outgoing packets from the daoc box, and that's it.. You should be well prepared now
