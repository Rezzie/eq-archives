This is how i have mines setup.. with 2 pc's.. 1 cablemodem and a hub..
here is how it

CableModem___Internet <---
|
Hub
|
----Win2k ipXX.XXX.XXX.XX|Sygate 192.168.0.1
|
----Linux Machine-ip192.168.0.5

1.Cat-5 Cable Goes From Cablemodem to uplink port in hub..

2.Cat-5 Cable Comin From Win2k Machines Nic Into Wichever Port on the hub.. Mines is plugged in port 4

3.Cat-5 Cable Comin From Linux Machine To Hub Connected at port 5 Really doesnt matter wich port ya plug it into..

3.My ISP doesnt give out static ip's so i have to share the connection.. I use Sygate to get this done. So my Win2k Machine has 2 ip's.. One from ISP wich connects me to the world.. And one for my linux Machine to use as a gateway.

Example.. XX.XXX.XXX.XXX is the ip from my ISP.. then sygate makes my computer 192.168.0.1 for gateway of the internal network.. basically sygate is a virtual nic..
So any other comp i hook up can make its own ip within the range of
192.168.0. Last digit can be 2 to 254..

In linux i just did ifconfig eth0 192.168.0.5 netmask 255.255.255.128 since im sharing the net connection and my real ip is on that subnet.

then i do the command route add default gw 192.168.0.1

and that adds my win2k machine as the gateway.. Do it in the same order..

My route table looks like this..

192.168.0.0 * 255.255.255.128 U 0 0 eth0
default 192.168.0.1 0.0.0.0 UG 0 0 eth0

OK basically the first ip is saying thats the local net.. hence it has a zero on the end.. So it has no gateway.. the U means it's up

default is pointing to the internet.. and in order for me to get there i need to pass through the gateway wich is my win2k machine.. Basically The Sygate Program is 192.168.0.0 thru 192.168.0.255 and U means its up and the G means its a gateway.

So basically when you do

iconfig 192.168.0.5 netmask 255.255.255.128 eth 0 up

thats basically putting that first line in your route file on its own.. and telling your computer its ip is 192.168.05 and to communicate on 255.255.255.128 as a subnet or a netmask or whatever..

and route add default gw 192.168.0.1

is just letting my computer know where to go through to get out into the open net i suppose.. i explained this in my own way.. because im no expert or nothin this is just how i got mines done. hope this helps ya out.. happy sniffing

Oh and in your showeq.conf file.. dont make it sniff 192.168.0.1 make it sniff your computers real ip.. Example i would have showeq sniffing the XXX.XXX.XXX.XX one or whatever..

The packets just have to be sent to the Linux box somehow. It will work by plugging both computers and the cable/DSL into a hub, but the problem with that is if you want to use the Internet on Linux you need 2 IP addresses OR you need to have the hub behind yet another computer not running the EQ session you want to sniff, which is not really worth the trouble. The firewall and router on the EQ computer will also be pointless since nothing will be behind it. If you don't care about using the Internet on Linux or you can get another IP, using the hub will be the easiest method.
