gregchant
Zombie 		posted 02-04-2001 02:20 AM            
No, that will not work Bobber9000. The way client-server interaction works with items is this:
1) Client requests item with id####
2) Server checks to see if item #### exists.
3) If item #### does not exist, return nothing and log for possible malicious usage.
4) If item #### exists, return item information.

The editing of the NO-DROP tag was client side, and allowed you to change the no-drop tag to lore before the server verified the item, allowing you to drop it. Once dropped, the server resets the tag back to NO-DROP. However, one can pick up no drop items, allowing you to effectively give someone an item via the drop method.

However, they probably fixed this in one of two ways:
1) Dropped item cleanup was increased to an insane refresh. Basically you drop an item, it disappears almost instantly. (This occured, but unknown as to why they did it.)
2) They moved the offset for the NO-DROP tag (most likely)
3) They increased the validation timer to check basically every time you do anything with an item (not likely, as it would add a huge burden on both bandwidth and server usage)
