Vexorile: Yes, you can do that very easy. Both Mandrake 8.1 (http://www.linux-mandrake.com/en/81.php3) and RedHat 7.1 (http://www.redhat.com/download/howto_download.html) both come with simple GUI install systems which do all the work for you. They both feature a PartitionMagic like program built-in to the install, to easily resize, move, add, or remove your partitions. Using that you can make a Linux partition with a few clicks of the mouse. After the install when you boot up your computer you simply chose what OS you want to boot up into, Linux or Windows.

You could also use Mandrake's Lnx4Win which enables you to run Linux on a Windows patition, so you don't need to partition your hard drive at all. However it has slightly less preformance.

Down_System: You do not have to run fdisk at all. As described above there are simple GUI partition managers which come with most Linux distrobutions, including RedHat, Mandrake, and others.

Les: Yes, you are right. You have to download at least one 600meg ISO image, and there are also two extra optional disks with extra programs, both about the same size. You also need a CD burner to burn those onto a CD.

You /could/ download all the RPM packages, however it would be a real pain, since there are thousands of packages to have to download that way. If you don't have a CD burner, or if you don't have a fast connection and don't want to wait I'd suggest going up to the store and buying it. Mandrake and RedHat are both around $10-$20 for the basic versions in the store. It can get higher for "deluxe" versions though.


I suggest you take a look at http://linux.com/learn/install/ for some more help.

A server-class installation is most appropriate for you if you would like your system to function as a Linux-based server, and you do not want to heavily customize your system configuration.

Warning

During the server-class installation, the X Window System is not configured and no GUI will be loaded when the system boots, unless you choose to install the appropriate packages during package selection. A 256 MB swap partition.

A partition of at least 512 MB (mounted as /usr).

A partition of at least 512 MB (mounted as /home).

ok, you manged to install rhat with only 100 megs downloaded, i'll give that one to you. But at 100 megs, your likely to want to install something else in the future, and when it comes to post-install software additions, apt blows away rpm.

Redhat example:
Base + apache installed, you decide you want to add php and postgreSQL support (keeping it small)
1) go on the net (if you want new versions) or your CD (old versions) and find the porper php and postgres RPM's.
2) attempt to install them, and find they need dependancies
3) identify deps and go back to step 1-2 until you have them all satisfied.
4) commence installation
5) done

Debian example
same software
1) type "apt-get install php4 postgres"
2) done.

The difference becomes very noticable when you get into bigger deps (ie, apt-get install gnome vs finding all its dep rpms)
That, and a daily apt-get update; apt-get dist-upgrade will keep your entire system up date with that days package additions (things like updates to gcc the day they go live, and often sooner with dev snapshots).

mandrake is good for noobs, slackware is for really learning linux and debian is your reward for knowing what you are doing.
