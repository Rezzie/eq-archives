I dunno what features ShowEQ really contains any more, since I don't run it regularly, but here's some things it didn't have when I started working on SINS:

- alerts/filters editable from dialog inside the application

- lots of map drawing optimization
- spawn list doesn't jump around
(these 2 items make SINS usable on a P100 with 64M of memory -- ShowEQ was really hard to use on something that tiny because the spawn list constantly jumped around and repainted, and the map drawing code was inefficient - SINS can do something like 15FPS on a zoomed-out Dreadlands map on a Celeron 600, and, more FPS than you'll ever need on something faster)

- multiple maps
- focus on selection (map follows a mob around)
- spawn point tracking/timer (automatic, not player specified... it tries to discover spawn points itself instead of relying on the player to specify them)
- easy access to common settings by right-clicking
- drop window
- a LOT fewer menu items (I didn't figure out all the ShowEQ menu items until I started looking at the code
- customizable map colors/font
- "radar" style display on the map (with names above the dots -- use the comma and period key to change your player's radar distance)
- map FPS control
- spawn list FPM control
- enhanced depth filter controls
- zoom level control on map

The only reason SINS exists is because I started by rewriting the guts of ShowEQ as far as keeping track of data is concerned, and then it became really hard to backport these visible GUI features back into ShowEQ without also having the guts in there.

That's just a list of some things I can remember at the moment.

I'm probably going to add a spell tracker to it like the one that just went into ShowEQ.
