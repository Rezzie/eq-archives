> hmm, ok downloaded EQInside and zoneconverter from hackersquest. Got
> cabeast_obj.wld out of one of the s3d files. Imported it into ZC and then tried to
> "Export to SCR" results in "Nothing to Save". Same thing with export to obj.

Don't get your .wld file from the _obj or _chr file -- for ZoneConverter you want to go into cabeast.s3d and extract cabeast.wld. Otherwise carry on as you were and it should work.

I speculate that the _chr file contains characters (mobs) and _obj contains inanimate objects (trees, signs, etc). It'd be nice if ZoneConverter would work with the _obj's .wld file in concert with the standard .wld file and place the objects where they go.
