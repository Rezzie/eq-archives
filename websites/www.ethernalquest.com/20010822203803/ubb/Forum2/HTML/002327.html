Yes, we are designing the new EMU to run on linux and no there isn't one availible now.

In fact, after evaluating a lot of different alternatives for implementing this, we have (mostly) decided on using some classes that I wrote some time back. I'll be rewriting them and they will be LGPL (well, they are now in fact) and they will support threading and thread synchronization in an abstracted manner on both Win32 (both with and without MFC) and on linux under gcc.

In fact, this seems to have moved up slighly in importance, so it's possible that we'll have linux support in the first release.
