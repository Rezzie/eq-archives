It wouldn't be too hard to build a program to remotely control eqinside. It could then loop over all the contents of an s3d file and dump them to some common directory.

Here are some thoughts I'm working through on how connect the models:

HARD-CODED
The sprite def labels each vertex with two numbers. (in the 6th and 8th data segments) The 8th could potentially label a body part (8 = left & right upper arm, 9 = torso, etc) The 6th seems to label points within a part.
For example the torso contains a bunch of points labelled 3
The left upper arm has a bunch labelled 3, and another group labelled 4
The left fore arm has a bunch labelled 4 and anothe set labelled 5
The left wrist has points labelled 5.

Perhaps the client hardcodes "rotate points 3 in upper arm to connect to points 3 in torso" then rotate points 4 in forearm to connect to points 4 in upper arm", and so on. This same style seems to occur in the non-human models as well. I have also looked the "large rat", and found the exact same format of data.

MESHOP
The only part of the format I don't understand is likely the "MESHOP" portion, so that just has to be data about how to connect the model together. I say "likely" the MESHOP portion as I have been using the binary data in global_chr.s3d and don't have any of the ascii fields that jellognome mentioned in his original post about character models.
I noticed that the first word of each op is a uuually number from 0->num vertices in the model. Perhaps the second two words are operations to be applied to that vertex?
Occasionally a number way outside the vertex count is in the first word, perhaps these are mesh wide operations?

TRACKDEF
Another possibility is the "TRACKDEF" and "TRACK" files. I notice a lot of them have names like "ELF_CH_POINTDEF", ELFR_POINTDEF" Perhaps these "POINTDEF" give information about where the parts should be attached? I haven't spent much time exploring this as I have been focusing on the meshop for answers.

Anyways . I would love to work out how the data is stored (including the animations eventually) and I would be happy to work together with you to work all this out.

I'm going to continue exploring the meshops as possibilities. However, more information would be useful. Does anyone have ascii versions of the models? From the posts I've seen online about this it appears some people do have ascii character models. Is that what is stored in chequip.s3d?
