First a quetion: Why? What good would it do you?

Now the answer to your question: No. When your computer communicates with another computer on the Internet, your IP address is absolutely required to send information to you. Hiding your IP from the login server would be like placing an order from an on-line store and not giving them your address. They'd get your order but they wouldn't have a clue where to send the package to.

As for linking accounts, I think they go by CC#. Since IP addresses are often dynamic, and some ISPs use NAT/PAT, IP tracing isn't really a viable method to use. One day I could be on EQ and do something SEQ related to get banned, and because you had the SAME IP the day before they think it's one user and ban both. I think not.

Although it might be possible to set EQ up to use a proxy of some sort, it would be difficult if not impossible to set up, and wouldn't help you any unless you used a public proxy. And if you used a public proxy that doesn't log who goes through it (yes, proxy servers CAN do that) then VI simply has to add a rule to a router that says "deny xxx.xxx.xxx.xxx" where the x's are the IP of that public proxy. And from what I remember about the cisco router programming it is literally one line that can be entered in mere seconds. You don't even have to edit a file and reload it. You just type the deny command.
