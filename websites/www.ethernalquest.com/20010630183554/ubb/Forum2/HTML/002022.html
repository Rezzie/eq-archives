There is something I would like to add to Alvec's FAQ doc file, is make sure you maintain Directory structure on extracted files from .zips!!!

1. make FAKE EQ folder and copy EQ files in there.

2.Extract Emulatorserver.zip INSIDE FakeEQ folder. NOW!!! This is where I got confused, Emulator server will create its own folder inside FakeEQ and this is OK! Don't get confused but the fact that folder name is ETHERNAL! My problem with runing Emu was that my FakeEQ folder was named Ethernal and when unziping created another "ethernal" folder inside first one, I foolishly trasported all those files inside first folder and screwup up default path.

3. As you continue with steps discribed in Alves FAQ, remeber that after pathed to Verant with everquest.exe, pathed with eqfiles.zip, you have run in folowing order:
a. Loginser.exe (inside Ethernal folder)
b. Worldser.exe (inside Ethernal folder)
c. Zoneser.exe (inside Ethernal folder)

When you run these files, evryone of them should open MSDOS windows and STAY OPEN. If that doesn't happens (MSDOS blinks infront of you and dissappears, then your server ARE NOR FUNCTIONING) Mine didn't cause I copied all Emu files directly into FakeEq folder.

D. Run Emulog.exe (inside FakeEQ folder)

hope this helps on part of possible problems
