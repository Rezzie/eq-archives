Zombie 		posted 05-31-2000 02:58 AM            
Since I'm no longer playing eq I'll post the source for my dinput.dll macro program. Keep in mind all the macros are hard-coded so you will need some knowledge of C to make your own unless someone turns it into a real macro program. Also the autobeg macro does work but it will sometimes cause eq to start acting funny (won't accept some keyboard input and other strange things) so it's disabled.

I'm guessing that if eqmacros works by putting dinput.dll in the eq directory Verant will fix this soon. You could probably use it safely if you put the modified dinput.dll in windows\system but that could create some problems with game installers, etc. The best thing to do would be to use a loader program that will restore the original dinput.dll when eq is done.

ps. Dll wrappers also provide a nice and easy way to do memory hacking (fullbright anyone? ).

On with the source (moved to geocities):
http://www.geocities.com/justsomedude3/dinput.c http://www.geocities.com/justsomedude3/dinput.def.txt

You need to link with dxguid.lib and change dinput.def.txt to dinput.def and add it to the project.

Zombie 		posted 05-31-2000 10:35 PM            
Ya that's pretty much how it works. The mouse macro is just a basic thing I used to buy bandages, food, etc. Every time you press CTRL P it will add 20 left button clicks and CTRL C will cancel all scheduled button clicks. To use it just position the cursor over the purchase button and press CTRL P for each stack you want to buy.

The only problem is the button clicks have to be delayed a few frames for eq to recognize them (depending on how fast or slow your computer is going at the time). For me 4 frames worked pretty well as long as there weren't a bunch of people standing around.
