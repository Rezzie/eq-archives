the reason the guy put SINS onto sourceforce was so that the showeq developers could have a look see at what he did. at first they wanted him to do a diff (which would have shown all the differences between the two) but he said it was WAY to big for that (lots of new files, totally redone structures) so he was going to find someplace to put the code so that the seq developers could see it. what better place than sourceforge eh?

i decided i wanted to see what changes he had made, and i must say it is pretty impressive. i personally prefer the 3.0beta myself, and will probably continue to use it (more out of habbit?).

i think that it would be well worth it to try to incorporate some of its features into seq (like the filtering looked a little easier).

i read a few places that a LOT of the structure in the back end (hey, i'm not a programmer...) is different... i dunno if it was any faster, but it sounded spiffy :)

just if it helps anyone, i modified my update.showeq.sh script for sins (its a little different on cvs than seq)

#!/bin/bash

cd /tmp/SINS
rm -Rfv *
cvs -d :pserver:anonymous@cvs.sins.sourceforge.net:/cvsroot/sins login
cvs -z3 -d :pserver:anonymous@cvs.sins.sourceforge.net:/cvsroot/sins co sins

cd /tmp/SINS/sins
cvs update -P
