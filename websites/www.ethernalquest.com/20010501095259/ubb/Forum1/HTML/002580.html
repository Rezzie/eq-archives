servername.everquest.com is prob an alias to the world server for that world. Each world is a world server and a cluster of zone servers, which is why you see eqzone35.blah in one zone and eqzone21.blah in another. Not sure how many zones per server.

The multiple address for patch.everquest.com are due to VI using roundrobin DNS, to load balance patching across multiple servers, transparent to the patch software or the user.
