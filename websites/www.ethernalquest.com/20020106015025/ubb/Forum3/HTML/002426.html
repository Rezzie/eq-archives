Any developer comments on this?

This is my current files which seems to be reliable again (libEQ change provided above as the struct changed from 10 bytes shifted).

One thing of note:

It seems the hps are rather complex now.
In the spawn structs, there seems to be single byte hp values (reliable when viewed as percent 0-100) for PLAYERS and MOBS. The spawn hp update packets still have 2 byte values, and player information is quickly updated with this information.

This causes a problem for Showeq as it uses the old 2 byte storage area from spawnStruct to internally store the current and max hps. I had to "borrow" an unused area of the packet struct (read not presenlty decoded, but seemed in my testing to always have 00 00 00 00 in the area) to save the hps and use a one time copy from the wire (over the net) representation to the final resting place the the QT libs use to print the hp when updating the screen. A complicated hack. A real solution would be to store these values outside the struct.

The max hps value is now the LOW byte of the word holding the light information. Light information is correct. This max hp value seems to always contain 100 decimal for players and spawns.
The old "maxHp" is now the currentHps but the HIGH byte has random values (I am not sure what they are, but they do not seem to colate with hps in any way I can tell).

GuildID does seem to have a unique value for each guild. It seems to be FFFF when 'unguilded'.

The WEAPONS struct value (from equipment[9]) only had about 13 possible representations. I assume they wished to include more than 13 new weapon images in Sol so it was enhanced to be WORDS instead of bytes. This is easy to represent in our
struct and I am sure was easy on their side as well.

For me, it seems to now be stable (with the exception of sometimes never being able to decode the encryption). I will be looking for alternatives to libEQ.a file.

Festerit

Size of working libEQ.a on a windows box is 4648 bytes, on a linux box it is 4636 bytes.

To get the new CVS, from a root prompt:

# cvs -d :pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq login
password:<enter>
# cvs -d :pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq co .
# cvs -d :pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq logout

Then compile it by:

# cd linux
# make -f Makefile.dist
# ./configure && make
# make install
