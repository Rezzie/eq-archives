Basically this is how it works:

1) You only need 2 computers, 1 with Linux the other with Windows.

2) Active net connection must be on the Linux box. If you have DSL or cable you will need to set the Linux box up with all the DNS, Gateway and IP addresses assigned to you by your ISP.

3) Your Windows will have the same settings as your Linux box for DNS, Gateway and IP address as the Linux box is merely acting as a transparent proxy.

4) IP Forwarding will need to be enabled on the Linux box to pass packets to all other computers on the network.

That wont work. SEQ is a packet sniffing program, so if the packets dont pass the computer's line, then it wont be able to sniff them. You would have to be using a hub for your setup to work and would have to go basically like this:

incoming line ----> into Hub
then lines from the hub into each cpu.
however, if your incoming line has packet encryption (pppoe), or your default gateway/router does not broadcast packets, then you will have to use your linux box as a gateway using ip masq.

What you can do with your setup as you ahve it now, is install the second card on the linux box, and get your connection to your isp working, then connect your windows box up to your linux box through the second nic and sniff packets off of it.
