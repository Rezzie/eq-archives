Well I managed to get my hands on a "backup" copy of Partition Magic 6, so at last I resized my NTFS partition to make room for installing Mandrake 8. It's definitely better than RedHat 7.0 mainly because it has updated versions of the kernel, XFree86, QT, KDE, and all that good stuff, so you get those pretty anti-aliased fonts right after the install. It was a major pain in the ass to get anti-aliased fonts on RH 7 :p Though I don't know what versions of things come with 7.1 so it might be different.

Anyway, I'm going to write what I had to do to get ShowEQ working on it. My install of Mandrake was a custom 1 CD. I selected every set of libraries to be installed except for "scientific workstation", but I didn't bother (de)selecting any individual libraries.

The first error I ran into was configure: error: Can't find X includes. Please check your installation and add the correct paths! So I downloaded ftp://carroll.cac.psu.edu/pub/linux/distributions/mandrake/8.0/i586/Mandrake/RPMS/XFree86-devel-4.0.3-7mdk.i586.rpm and did rpm -Uvh. You can use any Mandrake mirror if that doesn't work.

The next error was configure: error: cannot find pcap_open_live in libpcap. Read the INSTALL file. For this one you need two RPMs, ftp://carroll.cac.psu.edu/pub/linux/distributions/mandrake/8.0/i586/Mandrake/RPMS/libpcap0-0.6.1-5mdk.i586.rpm and ftp://carroll.cac.psu.edu/pub/linux/distributions/mandrake/8.0/i586/Mandrake/RPMS/libpcap0-devel-0.6.1-5mdk.i586.rpm and did rpm -Uvh libpcap*.

Next, I got configure: error: cannot find compresion library functions. Be sure libz is installed. This one was a bit decieving at first because it implies that you don't have the actual library installed, but the problem is actually missing headers; a zlib RPM was installed but not zlib-devel. So I downloaded ftp://carroll.cac.psu.edu/pub/linux/distributions/mandrake/8.0/i586/Mandrake/RPMS/zlib1-devel-1.1.3-15mdk.i586.rpm and installed it.

The last error I got was configure: error: libEQ.a not found. Be sure libEQ.a is already installed in /usr/local/lib or /usr/lib because I always forget to download libEQ on a system that doesn't have it ;)

Hopefully this will help out some people having problems with Mandrake 8 installs.

I didn't follow the guide on Mandrake, I went straight to what would be step 8, because I know how to fix any errors that come up. For people who have never used Linux it's better to get errors fixed before they happen. You might get errors that I didn't list in the first post in this thread because I did a custom install and selected almost all of the libraries, it might be different on default settings.

For the steps that only say to save the file that's all you need to do. You actually do something with a file in a later step, for example, you download a file in step 7b and do something with it in 7c. Try skipping steps 3, 6, and 7 completely and see if you get any errors on configure or make. You probably will get a missing libpcap error, but just use and rpm -Uvh the libpcap and libpcap-devel RPMs that are in my first post.

Zombie 		posted 08-18-2001 03:33 AM         
./configure && make is better to use since if either program bails on errors the other step won't be run.

./configure; make just tells bash to run one command then the other regardless of the success of the programs and is not good to use if one has to succeed before the next is to be run.
