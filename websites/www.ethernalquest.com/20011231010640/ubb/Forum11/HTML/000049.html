The rpm -e command is used to remove any versions of qt that you may already have installed. I think you get the message that no packages specefied to be uninstalled because none were found. Either way, I got the same message and everything worked perfectly.

Here is how everything should work:
cd /usr/lib/ - Start in your lib folder
ncftpget ftp://ftp.trolltech.com/qt/source/qt-x11-free-3.0.1.tar.gz - download the qt package into that folder
tar xvvfz qt-x11-free-3.0.1.tar.gz
- expand into that folder, which will really expand into a subdirectory named like the filename
mv qt-x11-free-3.0.1 qt-3.0.1 - rename the folder to qt-3.0.1 (this should be where you get the directory you say you don't have)
cd qt-3.0.1 - change to the folder
run config and make and etc.

If you followed all of these steps, you should have the qt-3.0.1 folder in your usr/lib directory.
