Ok, here is what I did to get ShowEQ 4 to work for me:
.
Did a clean Red Hat 7.2 Workstation Install.. choosing default rpms..and KDE as default
.
after Linux 7.2 is installed and booted :
.
1. open terminal window
2. mkdir rpm
3. cd rpm
4. launch netscape internet browser
5. goto http://www.rpmfind.net/linux/C/redhat/7.2/en/os/i386/RPMS/
6. shift-right-click these rpms
(make sure you change download dir to /root/rpm/name of file)
.
files:
cvs-1.11.1p1-3.i386.rpm
binutils-2.11.90.0.8-9.i386.rpm
glibc-2.2.4-13.i386.rpm
glibc-common.2.2.4-13.i386.rpm
glibc-devel.2.2.4-13.i386.rpm
gcc3-3.0.1-3.i386.rpm
gcc3-c++-3.0.1-3.i386.rpm
libgcc-3.0.1-3.i386.rpm
libstdc++3-3.0.1-3.i386.rpm
libstdc++3-devel-3.0.1-3.i386.rpm
kernel-headers-2.4.7-10.i386.rpm
.
also these rpms: needed for showeq to compile:
libtool-1.4-8.i386.rpm
libtoos-libs-1.4-8.i386.rpm
autoconf-2.13-14.noarch.rpm
automake-1.4p5-2.noarch.rpm
zlib-1.1.3-24.i386.rpm
zlib-devel-1.1.3-24.i386.rpm
gdbm-1.8.0-10.i386.rpm
gdbm-devel-1.8.0-10.i386.rpm
.
7. after all is downloaded..
go back to your terminal window.. should still be in rpm directory
.
rpm -Uhv cvscvs-1.11.1p1-3.i386.rpm
rpm -Uhv kernel-headers-2.4.7-10.i386.rpm
rpm -Uhv binutils-2.11.90.0.8-9.i386.rpm
rpm -Uhv glibc-2.2.4-13.i386.rpm
rpm -Uhv glibc-common.2.2.4-13.i386.rpm
rpm -Uhv glibc-devel.2.2.4-13.i386.rpm
rpm -Uhv libgcc-3.0.1-3.i386.rpm
rpm -Uhv libstdc++3-3.0.1-3.i386.rpm
rpm -Uhv libstdc++3-devel-3.0.1-3.i386.rpm
rpm -ihv gcc3-3.0.1-3.i386.rpm
rpm -ihv gcc3-c++-3.0.1-3.i386.rpm
rpm -Uhv libtool-1.4-8.i386.rpm
rpm -Uhv libtoos-libs-1.4-8.i386.rpm
rpm -Uhv autoconf-2.13-14.noarch.rpm
rpm -Uhv automake-1.4p5-2.noarch.rpm
rpm -Uhv zlib-1.1.3-24.i386.rpm
rpm -Uhv zlib-devel-1.1.3-24.i386.rpm
rpm -Uhv gdbm-1.8.0-10.i386.rpm
rpm -Uhv gdbm-devel-1.8.0-10.i386.rpm
.
8. you should not receive any errors with the above.
when done double-check your install by: in terminal window.type
rpm -q gcc3 gcc3-C++ libgcc libstdc++3 libstdc++3-devel
should say all is installed. will say "not installed" otherwise your ok
.
9. download qt-x11-2.3.2.tar.gz from trolltech.com
in same terminal window. type:
wget ftp.trolltech.com/qt/source/qt-x11.2.3.2.tar.gz
.
10. after qt is downloaded.. in same terminal window.. type
tar -zxvf qt-x11-2.3.2.tar.gz
.
11. once done.. type
mv qt-2.3.2 /usr/local/qt
.
12. now type cd /usr/bin
.
13. you should be in the bin directory.type:
mv gcc gcc.old
mv g++ g++.old
ln --symbolic gcc3 gcc
ln --symbolic g++3 g++
.
14. now type cd /usr/local/qt
.
15. now launch your advanced editor.
.
16. open file /etc/profile in your advanced editor.
.
17. add these lines before the word "done" after the word "fi"
QTDIR=/usr/local/qt
PATH=$QTDIR/bin:$PATH
MANPATH=$QTDIR/doc/man:$MANPATH
LD_LIBRARY_PATH=$QTDIR/lib:$LD_LIBRARY_PATH
<empty line here>
export QTDIR PATH MANPATH LD_LIBRARY_PATH
.
18. save file close editor.
.
19. reboot computer
.
20. launch terminal window . type:
cd /usr/local/qt
.
21. you should be in qt directory. type
./configure -thread
.
22. once done type
make
!!!this make take hours!!!!!
.
23. you should have received no errors thru the whole process.
.
24. now download the libEQ.a from anyone of number of resouces.
once downloaded.. type
cp libEQ.a /usr/lib
.
25. now download SEQ 4.. in terminal window type
export CVSROOT= server:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq
cvs login
hit enter when prompted for password
cvs checkout showeq
cvs logout
.
26. type: cd showeq
.
27. you should be in showeq directory. type:
make -f Makefile.dist
.
28 type: ./configure && make
.
29. when done type:
make install
.
30. grats.. seq 4 is installed..
.
31. open advanced edit.. open file /usr/local/share/showeq/showeq.conf.dist
replace default ip with your ip
save as showeq.conf
.
32. close advanced editor.. launch terminal and type.
showeq
.
SHOULD BE UP AND RUNNING.. took me 2 days of hell but got it to work with this exact process..
.
Thank you, caligirl

If you're starting with RH 7.2 it's extremely simple...

Install 7.2, just install everything.

Go to the 2nd CD and install libgcc3, libstdc3 & dev, gcc3, gcc3-dev (5 rpms)

Then compile QT 2.3.2 (make sure you use gcc3 & g++3, set your environment up and stuff..)

Then compile SEQ 4.

If you're running on a really old installation of linux and you dont explicitly *KNOW* what you are doing then you probably aren't capable of upgrading all your packages to spec and you need to just format the HD and install a fresh copy of whatever your favorite distro is, stop complaining and deal.

Hope this helps!
