I never used the spawn filters/alerts until just recently (no sound card). From what I've read though this looks like it may be the "old" method described in some posts here and in the showeq.conf file.

If that is the case, I'd think your problem is that you need to set the UseOldFilters to 0 in the showeq.conf file.

If that's not the case then I've also seen some posts concerning the * and _ in the names. I know in the new method they are not used if you are just filtering for a specific mob.

I only use the filters.conf (newer way) and it looks like this for alerts:

[Alerts]
a skeleton
a werewolf

Also I believe it does alert on names containing partial names in the [Alerts] section. For example if you used "a were" instead of "a werewolf" it would alert on werewolves, werebats and anything else with "were" in the name.

Really the best way to set it up is just to figure it out using the information that comes with the source. All the .conf files (showeq.conf, filters.conf, etc.) have more than enough comments for you to get things going.

When you run showeq, check the terminal window and you'll see 3 files and whether showeq found them. showeq.conf, filters.conf, and spawns.conf. It will say NULL if it didnt find them. Copy this guys spawns.conf and put it in yours, then when you start showeq, use the -s /locationoffile/spawns.conf switch. Can also do ./showeq --? for all the switches.
