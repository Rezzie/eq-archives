Ok, I dont know if this is going to help but :

The first thing you have to do is be sure that you PCI vga card is not shring an IRQ.
The VIA chipset indeed has "issues". The best way to get passed that is to put the PCI VGA as far as possible from the AGP... last slot works for me.
For Windows 98, AGP has to go first in the bios. If you need extra IRQs, I suggest disabling Serial ports if you dont use em.
If everything goes as planned, you should have AGP as primary display and PCI as secondary.
Also, some PCI cards dont work in dual display, you might want to go for an S3... these are the only ones that seem to work for me.

Although D3D will work fine in such conditions, you'll have to disable the secondary display card in order to use OpenGL hardware acceleration.

I hope this helps, if you have other questions, let me know... I run 3 monitors on the same machine and I know how painfull it can be to get all that working. :-)

VIA sucks the major ass. And ATI cards do not work correctly on VIA chipsets. This is a common problem. I would suggest finding yourself another primary video card, maybe a Voodoo (Which seem to work alright with VIA chipsets) The init display first setting wont help the problem, it is just the card that the bios uses to show you its bootup and POST screens, if the bus you have set as "init display first" does not respond to video bios calls, the bios will try the other bus next anyway, it just saves a few clock cycles to have it set right if you only have one video card. ONE other thing, you are not supposed to use dual monitors with AGP/PCI mixed. If you must have dual monitors, I would suggest having both video cards be in the PCI bus.
