Zombie 		posted 05-19-2001 09:40 PM            
The best setup would be to put two NICs in your linux box and enable IP masquerading.
There is a very simple firewall script that you can install that needs a little configuration to make work.
http://www.redhat.com/support/docs/tips/firewall/firewallservice.html
Here is what I recommend.
Cable comes from wall to cable modem.
Goes from cable modem to first NIC in Linux box.
Comes out second NIC in linux box and goes to HUB.
Hook windows box to HUB.
Statically set your IP address of the second NIC in the linux box to 192.168.0.1
Statically set your IP address of the windows box to 192.168.0.XXX where XXX is a number from 2 - 254.
Statically set your default gateway on the windows box to 192.168.0.1
Set your DNS to whatever your ISP DNS is.
When you run SEQ, make sure you tell it to sniff the packets on the IP address of your windows machine.

I hope this makes sense.
Good luck.
Z

autosensing hubs are fine as long as both the EQ box and the SEQ box are at the same speed. Hubs which implement 10bT and 10bTX ethernet do so by createing 2 virtual hubs, a 10mbit one and a 100mbit one, and switch packets between the 2 virtual hubs.

in short, 100 and 10mbit in a hub == same as a switch. switch == bad for seq. dont do it

I would like to point something out that everyone is over looking. It is ok to use a switch and it will work with SEQ. The reason I say this is because if you have your linux box configured as your Firewall then your packets have to pass through the linux box by default to get to the internet. So if you are using a switch in this type of setup then it will work. Also If you are using the "Firewall" type setup you DO NOT need to put your NIC cards into Promisc mode.

I hope this helps clear things up.
