Zombie 		posted 01-25-2001 07:00 PM         
It would only be even remotely if all the following were true:

1. Only one person were attacking the mob (because you wouldn't know the damage that every other did unless they were all using melee.
2. SEQ recorded the exact time of the each attack from start to finish so regeneration would be able to be assessed throughout.
3. No DoT's were used since the exact amount of damage isn't sent to the client.
4. No diseases were used that would inhibit regeneration.
5. No pets were used (since some pets are casters and the damage wouldn't be reported).
6. The monster isn't a caster that can heal or buff itself.
7. The monster isn't near any friendly casters that would heal or buff the monster.

I'm sure I'm missing some.

But there is definitely some valid and useful information that can be acquired on the monsters. Here are some that I think would be useful.

1. How much it hits for and how fast.
2. What kind of spells it casts and how much damage it does.
3. How often spells are resisted and which ones are resisted (recording the level of the caster though)
4. All known spawn points and sightings.
5. Factions gained/lost (though I know that the exact amount would not be possible to track automatically)
6. Its class and level
