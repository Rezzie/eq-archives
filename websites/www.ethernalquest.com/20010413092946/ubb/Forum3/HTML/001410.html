I may just be missing it, but I don't recall seeing a clear description of the map file format, so I thought I'd post what I think I know and see if anyone could fill in the blanks.

There seem to be 3 main types of lines in the map file; the header line at the top, point marker lines, and the lines used to actually draw.

The header line looks like:
Name of plane to display,zonename,?,?,?,?,?,?

I'm not sure what the numbers (represented by question marks) mean. On the ones I've seen they're all positive, so I would guess they're not maximums and minimums...

Drawing lines:
?,name,color,numpoints,<numpoints*2 numbers, presumably coordinates>

I've seen L and M in the spot with the question mark, so I'm not sure what it's for (I was guessing L to indicate it was for drawing till I saw one with M). I've seen "line" and "line_name" in the 'name' slot.

Point marker lines:
P,name,color,coordinates
These seem pretty straightforward

So can anyone fill in some of the question marks?

it's been a while since i played with maps but here's what i have in a perl script i wrote to parse map files:

header:

( $map->{'p_name'},
$map->{'u_name'},
$map->{'x_center'},
$map->{'y_center'},
$map->{'x_scr_length'},
$map->{'y_scr_length'},
$map->{'x_map_length'},
$map->{'y_map_length'},
) = split '[,\n]', <MAP>;

hopefully it will be evident what each element is, because to be honest i don't use them in my scripts.

lines:

($abbr, $name, $color, $count, @points) = split '[,\n]';

if i remember correctly, the $abbr designates whether @points contains coordinate pairs (x,y for L) or coordinate triples (x,y,z for M). in other words, the number of coordinates given on an L line is evenly divisible by 2, while M lines are evenly divisible by 3.

hope this helps,

- fooble
