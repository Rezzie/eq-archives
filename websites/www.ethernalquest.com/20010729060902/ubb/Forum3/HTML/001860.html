in the world of X you have clients and servers.

An X server is responsible for controlling the display, io and everythign involved with the actual user interaction.

an X client is any program written to use X (ie xterm, mozilla, showeq, etc)

there is an enviroment variable called

DISPLAY

when you startx or login via (k|g|x)dm this variable gets set to :0.0 (desktop 0 on display 0 of the local machine).

whatever this var is set to is where xclients connect to be used.

setup xwin32 on a windows box. now your windows box has an Xserver. so on your linux box, right before you start an x application do

export DISPLAY=my-windows-box:0.0

now X applications are going to connect to your windows box on display 0, desktop 0. its as simple as that.

The only thing that is going to appear on your windows box are applications that you start after you set DISPLAY.


Alternativly (this requires a little more work and wont be described here) is the setup for linux box to allow X to remotly manage clients. Then you can get your windows box using xwin32 to request to be managed and you will get a graphical login screen jsut like you don on your linux boxes console. You can then login and it will load up your whole desktop environment, in a window on your windows box. From there you can start other programs, such as showeq.


have fun.
