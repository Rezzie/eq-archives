Innoruuk 		posted 03-08-2001 04:14 AM            
Ok, first let me clarify a few things. First off, no, there are no programs that allow you to view a monster's or corpse's loot before it is dead. This is because everything having to do with a mob's loot is server-sided up to the point you loot it; and even then the server just sends you whats on the corpse. However, as you are aware, mobs are able to equip different weapons. Sience we are able to view the equiped weapons in game, ShowEQ is able to find out what model number the weapon is. But because some weapons share a model number (IE rusty short swords and fine steel short swords all look the same when equiped), for the most part, it is practically impossible to determine the exact loot off of common mobs. This is different for rare mobs. Sience the weapons rare mobs drop usually have unique model numbers (for example, nothing else in the game that I can think of looks like, say, Braided Thulian Claws), we can usually determine what weapon they have, if any. While it would be nice to see a mobs loot before the kill, I am sorry to say it is not possible.

Froggy, you couldn't be more wrong. Loot is determined at spawn time. If it were at kill time there would be anti-farming code in place. As for sniffers showing it, yes, SEQ can display any loot that has a visible effect on the world (ie. GLS on a wisp, short sword on ghoul lord, etc...). But unfortunately, any loot that does not appear (like coin, or it's "head",) cannot be detected by SEQ or any other program that is not running directly on the EQ servers.
