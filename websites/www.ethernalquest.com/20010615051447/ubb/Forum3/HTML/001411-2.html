Zombie 		posted 03-31-2001 02:27 AM         
Chode, I suppose you have libpcap installed because the pcap.o file would have never been built if you didn't, so all I can think of is that you're not completely typing in the command. Make sure you're using the -lpcap flag especially in the c++ command.

Grant, if you're typing everything in correctly, you probably had problems during the make command (other than the link errors) for the files to not exist. Try running make again and see if you get errors while building one of the .cpp files. That or you missed a cd command somewhere.

rudyfink, you're probably missing the gd libraries that draw GIF files. Get gd-1.8.3-4-i386.rpm and the others it requires from http://www.rpmfind.net if you want an RPM, however, I don't know where you could download the source for it. You may actually need to install more than just the gd lib, but you'll have to try for yourself and find out if you do.

Zombie 		posted 04-04-2001 11:02 PM         
Facer, looks like you don't have GCC installed. Get an RPM from http://www.rpmfind.net or ftp.redhat.com or if you don't want an RPM go to http://www.gnu.org/software/gcc/gcc.html

Mikeb4kool, you have 2 options. One is you can plug the DSL thing (it's not really a "modem" per se, sorry, it's a pet peeve ;)) into the uplink port of a hub and plug your Windows and Linux boxes into the hub. Then have ShowEQ sniff the IP of your Windows computer. The other option is to plug the DSL thing into the Linux box, setup IP Masquerade, and connect the Linux and Windows with a crossover cable or hub. This is harder to get set up but you have the advantage of being able to use one IP address for two computers and you can set up a firewall for security.

losshadow, I don't know how long it would take on a 233, but 15 HOURS sounds a little extreme. It takes like 20 minutes to build on my P3 500. The longest step after that is building ShowEQ itself, which takes me about 5 minutes. That would translate to what, 3 and a half hours on there? :p I don't know if that's normal but if it's not I have no clue what the problem could be.

Zombie 		posted 04-06-2001 09:13 PM         
About the missing .o files, you either didn't do a step somewhere or one failed. I have no idea on fixing segfaults at startup or doing an audio file alert (my sound doesn't even work I have to use the system bell). Mobs are unknown until ShowEQ gets the decryption key, which happens whenever something spawns or someone zones in. This should not take long, and usually happens within 5 seconds, unless you're like the only person in the zone. To rebuild 27-29 should do it but you may want to configure just in case. You can just make a sh script so you don't have to type in all of it. You can get info on CVS from its man page (Do man cvs in a console window). Yes you would want to start at step 16, but note that if you have a QT 2.0.1 RPM already installed you can eliminate all steps from 22-30 except 23.
