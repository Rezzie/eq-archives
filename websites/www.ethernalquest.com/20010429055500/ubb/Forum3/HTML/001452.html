Documentation on the filters is sparse to say the least. You don't need to use any command line settings for filters, just like all the other features. All settings are in showeq.conf, and you can run showeq with /usr/local/bin/showeq and no parameters. For example in my showeq.conf there is the line SpawnFilterFile /usr/local/share/showeq/filter.conf. It defaults to the same thing except "filters.conf" but for some reason this wouldn't work unless I renamed everything to "filter.conf" The idea behind the filter file is that for filtering you put a regular expression matching whatever under the appropriate [Name]. Like if you wanted to filter out everything that's level 10 somewhere your filter file would look like
[Level]
10

Now, I don't yet know how exactly you'd filter everything less than or more than a certain number. Technically I know what regular expressions would do it but for some reason I can't make it work with some perfectly legal regular expressions. I'll post how to do it if/when I figure it out.

Setting alerts is the same idea except everything goes under the [Alert] category in the filter file. So if you wanted it to alert whenever Nagafen is in the zone part of your filter file would look like
[Alert]
Nagafen

If the spawn has "two" names (as it appears in the EQ client) you separate them with an underscore (or just put one of the two names if it's unique). It's also not case sensitive (by default; you can change this in showeq.conf) so if you wanted to alert for Vox you could either write "vox" or "Lady_Vox" or "lADy_vOX" but not "Lady Vox".

This is about all I can think of covering until I figure out how to get these regular expressions working if you have any other questions ask them.

You want to filter out everything levels 1-5.
Regular Expression:
^[1-5]$

Example:
You want to filter out everything levels 10-29.
Regular Expression:
^[1-2].$

Example:
You want to filter out everything levels 15-18.
