He didn't have to put it in such a harsh way, but you would be able to find the answer if you looked in the threads on the front page, you don't even need to search for it :p The answer for all of the questions asked in this thread are on the front page. That's why he was pissed, it's just so easy to find the stuff but people don't. But I'll be a nice guy and answer.

No, it does not need to be the gateway, but it must see the packets. This usually involves plugging the ShowEQ and EQ computers into the same hub (hub, not switch) if you don't want to use it as a gateway. However, the gateway method is the preferred way. If you have a 10/100 hub, the NICs on the EQ/SEQ computers must be both running at either 10 mbps or 100 mbps. If one is 10 and one is 100, it probably won't work (though it might), especially if you have a cheap hub. And don't come back saying "I can ping between the two computers and ShowEQ doesn't show anything" because being able to ping means nothing other than you have them hooked up to the network. :p

Your NIC will automatically be put on promisc mode.

The fix for ShowEQ on test has been out since the day after it stopped working. I will bump my thread describing how to fix it, but again, you could have easily found it by browsing the front page :p

You can get a tarball of the latest CVS version from http://www.krisp.com browse around for it on the site.

ShowEQ can run on any distro, but it's going to be easier to do on some than others. Some people have had to install things on Mandrake, so if you get a problem, search the board. RedHat would probably be a little bit easier than Mandrake, just because I hear more reports of people needing to install or upgrade things with Mandrake.
