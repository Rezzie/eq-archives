Ok anyhow, since it seems nobody cared to read my post (which was encoded in binary), I will tell you about what I was saying.
I made a program to translate instantly between hex<-->ASCII and binary<-->ASCII. Now here are the pros and cons for it...
Pros:
1. It "searches" for binary and hex in what you input. So if you type in this (ASCII-->Bin):
asdlkj 23lkj 09alkj00101100j akjs alkjas
It will output this:
,
This is because it will search through all the other crap you put in there until it finds valid binary, and then it will only translate the binary.
2. It is fast, small, and doesn't use up too much RAM. The program is a mere 84 KB. I pondered for a long time over the algorithims to use in it, and I finally came up with some very efficient ones. There is no delay between your input and the translation. It happens extremely quickly.
3. It is FREE.

Cons:
1. It has a crappy user interface. It is a DOS type progam. I built it rather quickly, mostly thinking about what algorithims to use, and so for now it doesn't have a nice interface.
2. It can only take up to 200 characters for now. I can add more, but the more it can take, the more RAM it uses when you run it. This only really becomes a problem when translating Binary to ASCII, because the binary takes 8 parts for a single character... (and then if you put in spaces...)
