Zombie 		posted 12-31-2000 09:41 AM            
I seriously doubt if the choice of '#' was designed to mess with ShowEQ. It was probably just a convenient char to use.

ShowEQ uses the QRegExp class from Qt for regular expression parsing. This means you should be able to escape any char with '\'. (i.e. \#)

When ShowEQ reads the filters file, any line that starts with '#' or ';' is ignored. This char MUST be the very first char so '.#' (note the leading match-any-char wildcard) will be passed on to the QRegExp parser.

For more info on using the regular expressions look at:
http://doc.trolltech.com/qregexp.html

(This is for Qt 2.2, but they don't seem to have changed it since Qt 2.0.1)
