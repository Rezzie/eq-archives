Zombie 		posted 07-29-2001 04:29 AM            
There are a couple ways you could do it. If you want ShowEQ to include the name of the spawn in the email, you'll probably have to do some code hacking (or try to get someone to do it for you), but if it doesn't matter or you don't know C++ (or don't want to start asking people to code for you), you can do it just by editing showeq.conf. Open it up, and go down to the SpawnAudioCommand line. Right now it probably says something like "/usr/bin/esdplay spawn.wave &" but that can be replaced by any system command.

Here is an example where the email address I want the message sent to is adenine@adenine.yi.org and my SEQ computer's email is seq@adenine.yi.org (to be replaced) that should work assuming you have sendmail installed, but I haven't tested it so there could be something I'm overlooking.

File /usr/local/share/showeq/seqmail:
To: <adenine@adenine.yi.org>
From: ShowEQ Computer <seq@adenine.yi.org>
Subject: Spawn Alert
Look, something has just spawned!

In file /usr/local/share/showeq/showeq.conf:
...
SpawnAudioCommand "cat /usr/local/share/showeq/seqmail | sendmail -bm -t"
...

And it should send the email. If you want to hear the audio alert as well you can do something like
SpawnAudioCommand "cat /usr/local/share/showeq/seqmail | sendmail -bm -t && /usr/bin/esdplay spawn.wave &"

Zombie 		posted 07-29-2001 10:02 PM            
Do you have the command in quotes? If you do and it still doesn't work, you can put everything in a shell script and change the command to running it. Example:

File /usr/local/share/showeq/alert:
#!/bin/sh
cat /usr/local/share/showeq/seqmail | sendmail -bm -t
/usr/bin/esdplay spawn.wave

In file /usr/local/share/showeq/showeq.conf:
...
SpawnAudioCommand "/usr/local/share/showeq/alert &"
...
