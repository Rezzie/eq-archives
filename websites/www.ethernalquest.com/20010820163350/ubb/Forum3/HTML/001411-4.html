Zombie 		posted 04-24-2001 09:28 PM            
mebbekew, nope you're going to need to rebuild at least. EQ_Junkie, first find out what the error is on the c++ command if there is one. The problem could be the wrong libpcap. There's no distro-specific versions of libraries. A Linux library is a Linux library, the distribution doesn't make a difference. You can just get any version of libpcap, preferably, the one that comes with RH 7.0 since it'll definitely work. Just make sure you erase the one that comes with Mandrake or you could have a problem.

Kimper, (looks like you figured this out, I'll post just in case) make sure you have an XFree86-libs-4.0.x RPM (where x is whatever version you have, probably 1). The file libXext.so.x (where x is the version) should be in the /usr/X11R6/lib directory. The reason why you're not seeing data is that the router is probably acting as a switch. The simplest thing you can do is something like this:
Router (plugged into Uplink port of hub)
|
|
Hub--------EQ box (Windows)
|
|
SEQ box (Linux)

With this setup you don't need to change any router settings, just set SEQ to sniff the EQ box's IP (probably 192.168.something.something).

Anything in the previous guide doesn't matter. With this one you shouldn't get any errors. The problem is that some code was added that isn't supported by old versions of QTs, so basically as of now you can't build ShowEQ unless you have a later QT. This was supposed to be fixed before the new code went into CVS, but I guess it wasn't. As the guide is technically for RedHat 7.0 I didn't put anything in about upgrading QT since 7.0 already works. So until it gets fixed your only option is to upgrade to QT 2.3.0. You can either download the source and build it from ftp.trolltech.com/qt/source or get QT-2.3.0 and QT-devel-2.3.0 RPMs from www.rpmfind.net and install them.

Ok guys sorry I couldn't help earlier but I was at a concert all day today. I'm not sure why qt-devel wouldn't be installed. Epic, you're on 7.0 and not 7.1? Kanle, so you got the newer QT installed? I think I know what your problem is. ShowEQ is probably trying to use the old libraries on startup causing it to get the undefined symbol errors. To see if this is the problem first you need to find out where the newer QT libraries are. If you installed an RPM they are probably in /usr/lib/qt-x.x.x/lib (where x.x.x is whatever version you installed). If you installed QT 2.3.0, try doing this to run ShowEQ:
export LD_LIBRARY_PATH=/usr/lib/qt-2.3.0/lib:$LD_LIBRARY_PATH
/usr/local/bin/showeq
If ShowEQ now works, that was your problem, and if you say that was it I'll write how to permanently fix the problem. If that doesn't work double check that your new QT libraries are actually in /usr/lib/qt-x.x.x/lib. If they're not in that directory you have to change it to wherever they are.
