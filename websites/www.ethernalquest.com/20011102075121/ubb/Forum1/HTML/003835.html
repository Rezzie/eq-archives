Assuming you're running Win98:
===========================================
From System Properties (WINDOWS+PAUSE)

-> Virutal Memory
* Let me set my own virtual memory setting
Minimum: 1440 MB (2.5x your physical ram, you said 576)
Maximum: 1440 MB (2.5x your physical ram, you said 576)

-> File System
-> Hard Disk
Typical role of this computer: Network Server
===========================================
To go a step further, I suggest using Cacheman (instead of 'just' Network Server setting). It's free/shareware, easy to find via any search engine.

Then restart, scan disk, defrag, and give it a run. YMMV, but I've done this for four different puters that run EQ, and they all showed varying amounts of improvement.

Yup, that's your swap file. You can delete it, it'll come back every time you run an app that uses VM. Basically, assume that portion of your HD will always be filled.

Some folks with 768MB RAM turn off VM entirely and are fine with most apps. Haven't had a chance to try that with EQ myself. So long as the total footprint of used memory doesn't exceed your free physical memory, you shouldn't need VM.

For example, one game app I run never uses more than 520MB. So as long as I have enough unused physical memory, I dont need VM.

Zombie 		posted 08-30-2001 10:33 AM         
when you have 128mb or more of ram, 2.5 x ram = swap file is just retardedly to big.

for 128-256, i generally use 1x ram. when i go over 512, i go with 256mb of swap.

swap = fake ram on your hard drive... swap is SLOW. you do NOT want your system using the swap space at ALL. even unloadin infrequently used dlls to swap isn't such a great idea... i mean if you have 512mb ram, what is 1 or even 10mb of dlls, compared with having to do a hard drive read to get them?

your swap min and max should be the same, and you really should defrag your HD with a tiny swap file, THEN reboot... THEN make a larget swap file (i ususally disable swap, defrag, then create a larger swap file after the defrag is done and reboot).

the only thing slower than accessing swap on hard drive is accessing a fragmented swap file on your hard drive!
