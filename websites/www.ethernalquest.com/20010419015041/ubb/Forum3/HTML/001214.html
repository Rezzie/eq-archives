ShowEQ Networking Guide

This document was written as a guide to assist in the configuration of a home network. The methods listed below are by no means the only way to configure networks. This document assumes the reader has basic knowledge of computer, can install basic computer equipment and software required for it.

Dial-Up Connection to the Internet
If you are using a dial-up connection to the Internet there is really only one way to configure your network. You will have to connect to the Internet via the Linux Box.

Equipment needed:
1. 2 Computers ( One Running Linux, the other Windows)
2. 2 NIC Cards ( One for each computer)
3. 1 HUB and 2 patch cables OR 1 crossover cable
4. 1 Non-Win modem

Steps:
1. Install the modem in the Linux Box and configure it.
2. Install the NIC Cards. ( One in each computer)
3. Create a dial-up session to your ISP on the Linux Box. Redhat has several GUI Wizards that walk you through this process so I am not going to go to deep on that subject. Be sure you can connect and browse the Internet on the Linux Box before you go to the next step.
4. Either Connect the Crossover cable from the NIC Card in the Linux box to the one in the Windows machine or connect one straight through patch cable to each machine and plug the other end into the hub. It is not nessary to use both a crossover cable and the Hub, but if you only have 2 computers the using the crossover cable will keep you from having to buy the hub.
5. Give the NIC card on the Linux Box an IP address. For example we will use 192.168.0.1. In Redhat the command to assign this IP address to the NIC card is “#ifconfig eth0 192.168.0.1/24”.
6. Turn IP forwarding on. In Redhat the command for this is “# echo 1 > /proc/sys/net/ipv4/ip_forward”. That will tell the Linux Box that you want to allow packets to travel through the computer from one network to the other.
7. Now create the rule to allow the packets to travel from one network device to the other. Creating ipchains rules does this. From a command prompt in Redhat this is done by typing “#ipchains –I forward –s 192.168.0.0/24 –d 0/0 –j MASQ”. This command will allow any computer on the network with a 192.168.0.xxx IP address to go anywhere on the Internet.
8. Configure the NIC card on the Windows machine. Right click on “Network Neighborhood” and select properties. Find the TCP/IP stack that is bound to your NIC card and click the properties button. Select the option that says “Specificy an IP Address” and give it the IP address of 192.168.0.2 with a subnet mask of 255.255.255.0. After you have finished that click on the Gateway Tab. You want to give it the IP address of the NIC card on the Linux Box of 192.168.0.1 and hit the add button. Last click on the DNS tab and give it the IP address of a DNS server. If you do not know the IP address of one call your ISP they can give you theirs or you can use 24.4.48.33, 24.4.48.34, 24.4.48.35, 24.4.48.36. These are all working AT&T DNS server. You only need to install one of them, but I usually install 2 just in case one goes down.
9. Reboot the Windows PC and you should be good to go.


Connecting to Internet via DSL, Cable Modem, or LAN Connection with Firewall
There are a couple of different ways to configure your network to make ShowEQ work with a LAN Internet connection. The first one I will go over is the one I like best and that is setting up the Linux Box as a firewall. Doing this will allow you to add more computers to your network at home and put them on the internet without having to purchase more IP address from your ISP.

Equipment Needed:
1. 3 NIC Cards (One for the Windows Machine, 2 for the Linux Box)
2. 1 HUB and 3 Patch Cables OR 1 Crossover Cable.
3. Internet connection already in place (i.e. Cable Modem, DSL, T1

Steps:
1. Install 2 NIC Cards in the Linux Box and 1 in the Windows machine.
2. From the Second NIC Card in the Linux Box (eth1) connect a patch cable and plug the other end of it into the Hub or use the crossover cable and go from NIC to NIC.
3. Connect the other NIC card (eth0) in the Linux Box to the Internet using another patch cable and plug the other end of it into the Cable Modem, DSL Modem or what ever device you might have.
4. Configure the first NIC card in the Linux Box (eth0) with the settings provided by your ISP. In Redhat you can change these settings from the command prompt y typing “#ifconfig eth0 xxx.xxx.xxx.xxx 255.255.255.xxx”. The x’s will be replaced with data from your ISP. That will give the NIC card its IP address. Now we need to install the gateway. Again at the command prompt type “#route add default gw xxx.xxx.xxx.xxx”. The x’s here being the Default Gateway your ISP provides. You can also insert a host name here instead if you have the IP address in your /etc/hosts file. Lastly you need to give the Linux Box the address of a DNS server. Do this by editing the /etc/resolv.conf file. From the command prompt type “#vi /etc/resolv.conf”. This will open the /etc/resolv.conf in the VI editor. Now you need to add an entry if there is not one there. For example “nameserv 24.4.48.33” would go on the first line. Save the file and exit VI.
5. Configure Second NIC in the Linux Box (eth1) for your home network. We will assign it the IP address of 192.168.0.1. To do this type “#ifconfig eth1 192.168.0.1/24” at the command prompt.
6. Turn IP Forwarding on. To do this in Redhat the command is “#echo 1 > /proc/sys/net/ipv4/ip_forward”.
7. Because this type of configuration is always connected to the Internet I like to add TCP Wrappers as an added means of security. To do this edit the /etc/hosts.deny file. At the command prompt type “#vi /etc/hosts.deny” and add the line “ALL:ALL” to the file. To get these changes to take effect you will need to restart the inetd daemon on your system.
8. Now create the rule to allow the packets to travel from one network device to the other. Creating ipchains rules does this. From a command prompt in Redhat this is done by typing “#ipchains –I forward –s 192.168.0.0/24 –d 0/0 –j MASQ”. This command will allow any computer on the network with a 192.168.0.xxx IP address to go anywhere on the Internet.
9. Configure the NIC card on the Windows machine. Right click on “Network Neighborhood” and select properties. Find the TCP/IP stack that is bound to your NIC card and click the properties button. Select the option that says, “Specify an IP Address” and give it the IP address of 192.168.0.2 with a subnet mask of 255.255.255.0. After you have finished that click on the Gateway Tab. You want to give it the IP address of the NIC card on the Linux Box of 192.168.0.1 and hit the add button. Last click on the DNS tab and give it the IP address of a DNS server. If you do not know the IP address of one call your ISP they can give you theirs or you can use 24.4.48.33, 24.4.48.34, 24.4.48.35, 24.4.48.36. These are all working AT&T DNS server. You only need to install one of them, but I usually install 2 just in case one goes down.
10. Reboot the Windows PC and you should be good to go.

Connecting to Internet via DSL, Cable, or LAN Connection without Firewall
This type of configuration requires one less NIC card, but you will have to have one other IP address from your ISP. This type of configuration also leaves your Windows PC’s open to Internet attacks from the outside.

Equipment Needed:
1. 2 Valid IP address from ISP
2. NIC Cards (One for each computer)
3. 1 Hub, 2 patch cable and 1 crossover cable. If you have a hub with an uplink port then the crossover cable will not be needed.

Steps:
1. Because Cable Modems, and DSL routers are designed to plug straight into a NIC card they are built with a crossover on them. To reverse this so we can plug it into a hub we either need a crossover cable or a hub with an uplink port. Either take the crossover cable and use it to connect the Cable Modem or DSL router to the Hub or use a straight through patch cable and connect the Cable Modem or DSL router to the uplink port on the hub.
2. Install One NIC card in each computer.
3. Connect a patch cable from each of the two NIC cards to a port on the Hub.
4. Configure the NIC card in the Linux Box with the settings provided by your ISP. At the command prompt type “#ifconfig eth0 xxx.xxx.xxx.xxx 255.255.255.xxx”. That will give assign the IP address to the NIC card. Next we need to install a route to the gateway. At the command prompt type “#route add default gw xxx.xxx.xxx.xxx”. The x’s being the IP address that is assigned by your ISP. Lastly we need to add the DNS servers to the /etc/resolv.conf. Again at the command line type “#vi /etc/resolv.conf” and add the DNS server IP address provided to you by your ISP to the file. For example the first line should read “nameserv xxx.xxx.xxx.xxx”.
5. Because this type of configuration is always connected to the Internet I like to add TCP Wrappers as an added means of security. To do this edit the /etc/hosts.deny file. At the command prompt type “#vi /etc/hosts.deny” and add the line “ALL:ALL” to the file. To get these changes to take effect you will need to restart the inetd daemon on your system.
6. Configure the Windows PC with the information provided by your ISP. Right click on your “Network Neighborhood” and go down to properties. Find the TCP/IP that is bound to your NIC, select it and hit the properties button. When the next windows appears click the option that says specify an IP address and enter the IP address and Subnet Mask provided by your ISP. Next Click on the Gateway tab and add the default gateway provided by your ISP. Lastly click on the DNS tab and enter the DNS information provided by your ISP. Click the OK button.
7. Reboot the Windows PC.

A configuration you should also have...

There are firewalls you can purchase for reasonable prices now that can also be used.

If you are buying one, save money and get the kind that is single port (ie. DSL/Cable port and a LAN port).

The setup is similar to the "DSL/Cable/Lan w/o firewall" setup except you connect the firewall to the cable modem (straight through cable), the firewall to the hub (use uplink port on hub or crossover cable), following the same steps as the w/o firewall setup.

I am mentioning this because it is different from the firewall setup, since the firewall can't run SEQ!
