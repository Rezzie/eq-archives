anon04 - kudos for trying to help, but that the info you gave was really really weak... Obviously he's been to hackersquest.gomp.ch... he's at the forums, ain't he?

To Burningflame:

Answering your first question... The requirements are simple... two computers, one with some kind of Linux partition, and one with EQ... using the Linux as the gateway, or using a router... It is completely impossible by now, without severe reprogramming, to run ShowEQ on the same computer at EQ... Remember to have a router (so the packets ShowEQ reads get sent to all computers on the router)... or use the Linux machine as the gateway (Have Linux dialed up, or connected to the modem, so the packets pass through the Linux machine first).

For Linux help, a good site is Linuxnewbie.com They have a few walkthoughs, in-depth help, and can basically get you started with Linux... as for which Linux OS to get (You will ask it next... believe me.) I myself love Debain... but for a beginner, I would recommend either Red Hat, or it's beefy friend, Mandrake...

For more help go to the ShowEQ forums... they are smarter there.... or you can IRC to irc.trifocus.net in room #ShowEQ for showeq, and #HackersQuest for Emu help. (old emu)
Andon is one of the ShowEQ programmers, and he is ALWAYS on IRC in #ShowEQ... He's at the top. Just ask him for help, he should be able to. Tell him Icon sent you. If he is unavailable, just ask casey or krisp or Oakley or someone, and they should be able to. If you still can't find help, simply ask out loud a few times, and everyone there will try and help.

Answering your next question (second post)...
Minimize EQ Program? I'm Guessing you mean EQW, in which case, yes, that has to be run on the computer EQ is on. And run alone, it starts EQ all by itself.
Zoneconverter is meant to be run on the same machine as EQ also.

If you are asking if they can be run together, then the answer is yes again, Zoneconverter doesn't have to have EQ running to work, it simply extracts the zone files. (*.wld)

I hope that answered your questions.
