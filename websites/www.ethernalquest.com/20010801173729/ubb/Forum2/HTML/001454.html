great doc, Alvec!

one thing you fail to mention:

you talk of the #zone command but, by default, this command is disabled in the emu.. you have to do some hex editing to enable it..

a decent Windows HEX editor (actually, it can edit just about anything like Text and HTML and stuff.. it just does HEX too, it's pretty cool for what it is :) is UltraEdit from http://www.ultraedit.com

UltraEdit is trial for 45 days and then you have to register ($30.00.. /snicker) but remember, astalavista.box.sk can come in handy sometimes (not that i condone that type of thing ;)

pasted instructions on enabling the #zone command are from 1010101's post:

"Ok. To enable the #zone command, which will allow you to change zones while in the emulator, do the following:
First, open up ZoneServer.exe with a hex editor. It is located in your EthernalQuest folder.

Now, go to offset 00084364 (0001498C in hex) or do a case-sensitive search for "Ashran"

Next, overwrite "Ashran" with six null characters (00's) and save it.

Now, the next time you restart the server software and run it, you'll log into whatever zone you have it set for normally. Once in, type "#zone zonename" where zonename is the short name for the zone you want. You'll see a message saying you issued a gm command to change the zone. Now /q and log back in. You'll be in the new zone. This does NOT change the zone permanently (doesn't save to ZoneServer.js), so if you shut down the zone server and restart it, it'll revert back to whatever is set in ZoneServer.js

Have fun everyone."

maybe you wanna include something about this? people might get frustrated that the #zone command won't work for them.. :)

HTH
