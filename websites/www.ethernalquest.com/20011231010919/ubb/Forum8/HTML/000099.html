Story should probably come last. If I were you, I would work on getting everything up and running, then simply intergrating the story. Here's what I'd do (simplified)

1) get a small world built
2) code the minor things (walking about, talking, set up for the screen)
3) expand the world a bit, add structures
4) code more things in (npc's, animation, combat, ect)
ect.
ect.
ect.
until your game is able to run effeciently, then worry about a storyline.

Take things one step at a time, focusing too far in the future will lead you to failure.
