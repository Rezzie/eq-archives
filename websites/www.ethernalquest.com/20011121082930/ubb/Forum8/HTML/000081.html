I'm working on my own game. Nothing as advanced ( graphic's anyway ) as everquest, but it will be a MMORPG, with quests, spawning, trade skills, etc. It's in VERY early alpha... ( i haven't finished the login code yet, and the server's still buggy ) but it is going fast. Myself and my team will *hopefully* have it to where you can at least log in and move your character, though cheesy looking, around the map and chat with ppl.

For anyone that's interested, it will be 2d graphics, and the game is being done in VB.
( before anyone turns they're nose up at it, play it first. )

Imagine "The Legend of Zelda" meets "Final Fantasy 2" meets "Everquest". That's what I'm going for. Low overhead for the server and client, the multi options of everquest, and ( for simplicity ) battles akin to Final Fantasy.

Anyway, I WILL post updates on my game, and when we get to where you can login, I'll be putting up a website for it and *hopefully* have a main server. When we release the client, it's not decided if we'll release the server, but I'm leaning toward it. Reason...This is gonna be very easy to hack. If you wanna hack it, be my guest...I'll give you a server, make a world and hack all you want. Leave the main server(s) for the ppl that like Hack-free environment.
Thank you, and good night.

Hell, if I don't like your username, I'll ban you ( that's just an example, but you get the idea. )

As for saving userdata, our server generates individual folders in the application directory for user info storage. It is in "your" user folder that it stores all your different characters and they're info. Stuff like spells and items will be stored in a database, but we're not that far yet. I'll DEFINATELY need spell / item / skill input when I get that far. I'll also have to come up with a battle system, because I HATE ganking already existing battle systems...seems like cheating. ( again, that's a ways down the road. I just want to have a stable client/server relationship before I start on "game concepts" )

Adding in code that allows for a person picking up a weapon or using a tradeskill to create one will be easy once that's done.

Any more questions, I'll be happy to answer them. It is a very strong possibility that we'll opensource this, after the client/server is stable....but not before we get all the stuff done on it and have a working world. I may not opensource it because one of my goals is to have the server and client so versatile that all you need to do is have the devtools and build a world there without recoding anything...that includes spells and items and such. I don't want to hardcode any of that.
