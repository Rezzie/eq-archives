Well, after a few days of scouring the forums and finally succeeding in getting this great program to work, I thought I would share some of my experiences and observations for those having difficulties installing showeq. This is intended as a guide for a complete beginner, so sorry if it's simplicity annoys anybody. It contains information gathered from other posts on this board - the search function is a wonderful thing!!

If you do not yet have linux installed, then save yourself a lot of hassles and get one of the later distributions, Mandrake 8.0 is what I used, but from what I have read Red Hat 7.1 works equally as well. DO NOT use an older distribution than either of these - you will have to download and install heaps of rpm's just to get your system up to date so you can install showeq. Bite the bullet and pay the $25 or download it. I am not going to go into installing linux here - it would make this long post far longer, and there are plenty of resources around in other places on this.

Both of these are available as free downloads, just go the the relevant site (www.mandrake.com or www.redhat.com) and follow the links.

Once you have your linux system up and running, you then need to make sure you satisfy the pre-requisites for showeq. This is where most problems seem to arise for most people - if you don't have the correct packages installed, then it just won't happen. For Window's people, it would be like trying to open a game without the correct version of DirectX - it is just not going to work. For me, with a full expert install of Mandrake 8.0, all I needed to install was:

egcs-c++
libstdc++
libpcap
zlib

You can do this using the Software Manager included with the Mandrake installation - just type in the name of the packages under the "installable" tab, and they should come up in the window below. All of the files you require are on the 2 installation CD's.

If you have not done a full expert install, you may need to install the following files:

make
gcc
autoconf
automake
Xfree86 dev
libpcap
zlib
qt-2
qt-devel
gcc-1
gcc-c++
libstdc++
libtool
(taken from GameGuru's post)

If you try and do a make of the showeq files and your system does not fill the pre-requisites, the error messages will tell you. One thing here that can be confusing is that sometimes it will let you know it needs a file which may be included in a package with a totally different name - you can go to www.rpmfind.net and do a search with any file names you want and it should give you a list of the package which contains the necessary file. Like I said though, if you want to save yourself these kind of hassles, just use Mandrake 8.0 - everything you need apart from the showeq source is included on the CD's.

There is one file you will need to download - libEQ.a - you can find it at http://libeq.trifocus.net/libeq/i386/linux or http://www.krisp.com/new/libEQ.a.gz

This needs to be saved either of these directories:
/usr/local/lib/ or /usr/lib/
If you don't have this file saved in one of the above directories, then your installation won't work.

Now for the fun part - getting it up and running. This part is also a cut and paste of GameGuru's post with a few modifications.

!!Important - DO NOT use the version of showeq linked from hackersquest - below are details on downloading the latest version of showeq via a thing called cvs - nifty utility that will ensure you have the latest version of showeq. If you use any other versions, chances are it will not work at all, or it will work but your positioning will be all screwed up. Again - don't use an old version - use the one supplied by cvs.

Open up a terminal window (if you dont know what a terminal is, just click on you start menu and there should be a tab saying terminals, choose any of them - they are pretty much the same thing, they just look a little different.) You can also click on the thing the looks like a black TV screen on you menu bar.

Once the terminal opens up you just type in the commands as they appear and let linux do it's thing.

Type the following, as it appears - don't type the stuff in the brackets, they are comments!!

mkdir /root/showeq (hit enter)

cd /root/showeq (hit enter)

export CVSROOT=:pserver:anonymous@cvs.seq.sourceforge.net:/cvsroot/seq (hit enter - nothing will happen visibly here, it will just take you to another command promt)

cvs login (hit enter - it will prompt you for a password, but just hit enter)

cvs co linux (hit enter -it will download all of the necessary files into /root/showeq/linux/)

**The latest version of showeq (3.1.0) is still unstable - until a stable version is released, used the following command in place of "cvs co linux" above to download an older, stable version "cvs checkout -r v306-Stable linux" (enter (you don't need the quotation marks))**

cvs logout (you guessed it, enter again)

cd linux (enter)

make -f Makefile.dist (enter - you will see a heap of messages - don't worry about it as long as you don't get any errors!)

./configure; make (enter - this will take a while, depending on your system specs - took about 10 mins on my celeron 400! There are 3 error messages that come up early on through the install - don't worry, it is normal. If it does not complete, that is when you have a problem - read the error message and it should tell you that it can't find a necessary file: means you didn't satisfy the pre-requisites. Go back and double check you installed everyting from the list above.)

make install (enter - take a few seconds, again lot's of messages flashing past)

cd /usr/local/share/showeq (enter)

cp showeq.conf.dist showeq.conf (enter)

cp filters.conf.dist filters.conf(enter -I am not 100% sure, but I think this makes your showeq.conf file and filter.conf file)

You are done! Well, almost. You now need to let showeq know the IP address of the machine you run everquest on. You can find this out (if you don't already know) by clicking on the run command on the start menu of your windows machine. Type winipcfg and a small application will open up telling you the details of your machine. Type in the IP address that you find here, into the showeq.conf file. It is self explanatory where to put it once you open up the file and take a look. The showeq.conf file should be located in /usr/local/share/showeq.

If you have done all this, had no errors during your make and make install, then the program WILL work.

You need to make sure you can ping your Windows machine from your Linux machine - you can also do this from the terminal, just type "ping xx.xx.xx.xx (xx is the IP address of your Windows machine).

Start showeq by typing the following command: /usr/local/bin/showeq (again in a terminal window)

The program will now start, and it is time to fire up your EQ session and have fun!

If you are having problems (ie: showeq is showing you jack shit) you also need to make sure you either have a direct connection from your Windows machine to your Linux machine, then from your Linux machine to the internet (requires two network cards), or you have to have your machines connected via a hub - you can buy a hub for about $10 - $15 dollars, and network cards are equally as cheap. It will not work with a switch or a router (may be possible with a router if you know how to configure it).

The reason is, switches and routers only send traffic to the machine that it is supposed to go to, whereas a hub broadcasts traffic on every port.

Showeq works by "listening in" on the traffic going to and from your Windows machine and the EQ servers - so why it only works on a direct connection between the computers or a hub should be obvious.

Well, that is it for my idiots guide to getting showeq up and running. There is nothing new here - just bits and pieces taken from the real experts - I only installed linux so I could get this baby to run. Thanks to adenine! and GameGuru for their posts where you can pretty much find different versions of the above.

If you are having some other kind of problem, try searching around with the search button - but I can't see how you could be, because if you install Mandrake 8.0 and follow the above info, then it has got to be working!

Just a final note - you need to be logged in as root to do the install and also use showeq - most documentation will tell you to make another user without all of the permissions of root to avoid making configuration changes by mistake. While this is an excellent idea, it won't let you install/configure anything, and I believe you also need to be logged on as root to use showeq. Just type "su" in your console and enter your password if you are not already logged in as root.

Please feel free to add any corrections/comments/tips.

You probably have a switching hub - if you are getting no info in ShowEQ, this most likely means your linux box is not sniffing the traffic being send to your Windows machine.

From memory, there have a been a few posts regarding Netgear hubs (I think - do a search and you will be able to confirm this) in which people have described the same problem as the one you are having.
