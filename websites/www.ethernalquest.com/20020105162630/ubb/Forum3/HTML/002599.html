Recently Casting time and spell name have been incorrect in the Begin Casting Messages. This appears to be due to the fact that the packet containing opcode a920(BeginCast) has had it information moved a bit. This affects the BeginCastStruct structure within everquest.h . However upon investigation many other areas of SEQ use the structure. As a "Work Around" I am posting changes to the printf statements so that they output the correct information. If someone else feel inclined to actually fix this please do

Testing Status of Change: Moderate

File to Change: packet.cpp
Change to: case BeginCastCode:
Line Numner Approx: 2619

// float casttime = ((float)bcast->param2 / 1000);
//The line below is a replacement for the one above. Will correctly reflect packet change for casting time. Work Around for BeginCast packet change.
float casttime = ((float)bcast->spellId / 1000);
tempStr.sprintf( "SPELL: %s%s' - Casting time is %g Second%s", tempStr.ascii(),
// (const char*)spell_name(bcast->spellId), casttime,
//The line below is a replacement for the one above. Will correctly reflect packet change for casting time. Work Around for BeginCast packet change.
(const char*)spell_name(bcast->unknown0004), casttime,
casttime == 1 ? "" : "s"
);

emit otherStartSpellCast(bcast);
emit msgReceived(tempStr);

break;

Sorry this isnt in a diff format or such. This is a work around and I would not recommend placing this in a distro. A proper fix should be found.
