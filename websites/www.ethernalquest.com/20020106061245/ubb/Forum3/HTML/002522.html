There is a comment and a line in main.cpp (I was looking through the source a bit) which currently forces the client to not try to decode because of the current state of the library(libEQ.a). Here is a segment of the code from main.cpp:

/* Decoder override for coping with encryption changes */
#if HAVE_LIBEQ
//showeq_params->broken_decode = pSEQPrefs->GetPrefBool("Misc_BrokenDecode", 1);
showeq_params->broken_decode = 1;

if (showeq_params->broken_decode)
printf("Disabling decoder due to showeq.conf preferences\n");
#else
/* Default to broken decoder if libEQ not present */
showeq_params->broken_decode = 1;
printf("Disabling decoder due to missing libEQ.a\n");
#endif

The commented-out line to look at is:
//showeq_params->broken_decode = pSEQPrefs->GetPrefBool("Misc_BrokenDecode", 1);

To re-enable the use of libEQ.a for packet decryption, it seems you would uncomment that line, and comment out(or remove) the line following it, which is:
showeq_params->broken_decode = 1;

This line forces broken_decode to be true, no matter how your config file is set.

Hope this helps,
--Xel
