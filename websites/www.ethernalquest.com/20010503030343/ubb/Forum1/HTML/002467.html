It's all _documentation_, good _ideas_, and _examples_.

For programming in general, don't ignore the learning material that comes with the commercial programming IDE of your choice. I started with "that book that came with Borland C++ Builder." If you choose to program in C++, better choices that more fully cover the language and it's standards (praise The Standard) would be The C++ Programming Language, by Bjarne Stroustrup (creator of C++) and a good book simply on object-oriented design, such as <ok, I forgot the name of the one I picked up> by Booch. MSDN is a great free resource even if you're not programming for Windows, I would say. If you can deal with the concise and technical language used within it, the Standard C++ Working Draft is available free online.

Ideas, ideas, ideas! The language is nothing without a pretty damn spiff idea. This is where a lot of the success comes from.

Use the resources available to you. The internet brings access to help and examples illustrating the design & code for specific problems that you may face. The open source revolution brings even more access to knowledge to the learning programmer. Join or lurk about an open source project and learn from other more experienced coders. I like to visit www.codeproject.com for some reason, but there are a lot of places.

Mainly, find a programming language (I recommend C++) that clicks with you, and provides easy ways to program in the paradigms (procedural, object-oriented, rules-based?) you wish to use (which may be several, depending on each individual project). Get a book that you find is written in a way that is easy to read that covers the concepts of the language, and eventually get a more elaborate, reference-like book for all the low-level stuff that you may encounter.

"...To sum up, I recommend an approach that
– proceeds from the concrete to the abstract,
– presents language features in the context of the programming and design techniques that they exist
to support,
– presents code relying on relatively high-level libraries before going into the lower-level details (nec-essary
to build those libraries),
– avoids techniques that do not scale to real-world applications,
– presents common and useful techniques and features before details, and
– focus on concepts and techniques (rather than language features)." - Bjarne Stroustrup, from "Learning Standard C++ as a New Language", published in The C/C++ Users Journal and online at http://www.research.att.com/~bs/new_learning.pdf

Links:
- Bjarne Stroustrup's C++ page (has many useful links and FAQ's) http://www.research.att.com/~bs/C++.html
