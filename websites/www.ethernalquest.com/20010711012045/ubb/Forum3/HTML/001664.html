I have a DSL connection running through a LinkSys router coming into my windows box. To get my linux system to sniff packets on the network I needed to buy a hub because the Linksys router is essentially a switch.

Not knowing much about hubs I unfortunately bought a nice 10/100 hub (also made by linksys) that had auto-sensing in it. This I found out later is essentially a ghetto-switch box, and you won't be able to see packets coming in from your windows system.

Get yourself the cheapest hub that you can get your hand on and you should be ok. If it is above $30 it probably has some kind of switching technology in it.

I also have the linksys Cable/DSL router and have managed a work around. All you have to do is from the ADVANCED tab in the routers web configuration and choose FORWARD. Set the Packets to be forwarded 1-10000 and forward them to your linux box. Set your windows box gateway address to your linux box and on your linux box enable IP forwarding by typing the command line: ipchains -A forward -j MASQ -s 192.168.1.0/24 -d 0.0.0.0/0

This is assuming that you have an IP address in the range of 192.168.1.***.
