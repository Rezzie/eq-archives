You only need to keep legacy libraries installed, not the developement stuff. For example, you might have QT 1.x libraries in /usr/lib, and the headers in /usr/include/qt (as many distros do it). You install QT 2.x in /usr/local/qt, you just need to point to the right places. I.e., in /etc/ld.so.conf, put /usr/local/qt/lib at the top, so when you run ldconfig to update ld.so.cache, that dir is where any qt libs are found first. Also be sure to export (or set) your QTDIR environment variable properly (/usr/local/qt in the above example), and put the bin directory in your path (/usr/local/qt/bin). This is also why it is a good reason to remove any devel packages that are old, so older binaries are not found (like moc). For redhat/mandrake stuff, using RPM's, I believe there is like qt-1.xx package, and qt-devel-1.xx package, you would want to remove the devel one, and then install qt2 developement package.
