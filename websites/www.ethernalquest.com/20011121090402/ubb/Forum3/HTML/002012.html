Archaeopteryx
Decaying Skeleton 		posted 09-04-2001 05:50 PM         
When you installed VM ware did you select bridged networking? If so you are probably ready to go. You need to assign an independant ip address to the linux "machine" different from the host os. You mentioned you had a separate machine to run EQ on? There are two ways your setup will work:

1)If you use a hub, set showeq.conf to use the appropriate ip address (you eq machine's). You should be able to sniff your eq machines packets. Your done.

2)If you use a switch you have to set you Linux "box" to be the router. You will need two nics on the Linux "box" in order to route traffic from your Linux "box" to the rest of your network. Ill assume you use DSL or Cable Modem, connect it to the Linux "machine's" eth0. Assign the Linux "box" the internet settings your ISP assigned you, be it DHCP or a Static IP; make sure you assign those settings to your first ethernet device (eth0).
Next connect eth1 to your switch's uplink port. Connect your EQ machine to your switch. Now type the following commands from a termial window:

echo 1 > /proc/sys/net/ipv4/ip_forward

iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Of course, you have to setup your Win box to use the ip address you assigned the Linux "box" as a gateway, and obviously set the right ip address in showeq.conf

Hope this helps.

-Arch
