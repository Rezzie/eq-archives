Well, okay, so using the tutorial as a zone
explorer (with EZE) has a few issues. A
bunch of zones start you off in a GM teleport
box you can't get out of, or it's dark, or
the dude runs/swims to slow, or whatever.

After messing with my tutorial.exe a bit, I
was able to find a way to add a few more
features to it beyond changing the zone file
it loads. Go nuts...

The version of the tutorial I have shows a
Last Modified date of 05/11/1999, and is
1,845,248 bytes long. The EZE page
mentions other versions, so it would prolly
be a *BAD* idea to go patching offsets in
your tutorial file if your version isn't
the same.

You already know where to change the zone,
but just to keep all the info together, I'll
mention it again: There are 11 bytes starting
at offset 19797C where you can put in the
zone you want to load, so long as its
followed by hex 00 (NULL).

Change the race of the player at D511C. It
starts at 01 (Human), to make it Dark Elf
for ultravision, set it to 06. It will take
values 01-0C for other races and doesn't
know about Iksar.

If having to talk to the tutorial mobs
annoys you, get rid of them by patching
offsets D1818-D181C to all hex 90s.

Hex code 90 is the assembler instruction
"NOP" (a do-nothing command), so its used
to disable certain instructions.

You can have it report your current X,
Y,Z, coords, heading and some other stuff
in your message box:

Write hex 90 to offsets:
B3002-7,
B300F-14,
B30E5
Change offset: B2ECD to EB
Starting at offset B30D6, write:
0D A0 73 70 00
Change offset: B30DF to 94
Starting at offset B30E7, write:
90 E8 E3 9C 01 00

Note: if you combine this with the
flying patch below, it will make it a bit
harder to fly (you have to angle up more
to get off the ground).

If you're having trouble exploring the
plane of Sky or whatever, you can allow
the player to fly (like swimming, angle the
view up or down):
At offset D9F7E, change 75 to EB
Change offsets D9FE7-E8 to 90 90

Improve the swimming (and flying) speed:
He swims very slow, giving it a 4x increase
makes him swim a little faster than he
walks:

At offsets D9FF3-F6 write CD CC 4C 3E
At offsets DA003-06 write CD CC 4C 3F

Note, these correspond to the values 0.1
and 0.8. The first (at D9FF3) controls
the swimming horizontal speed, the second
controls the vertical speed. If you set
the second to some huge number, like 60
instead of 0.8 you will take huge dolphin
leaps out of the water! It's hilarious.
You won't be able to explore much underwater
though, since any move that goes up or down
will send you zooming to the top or bottom
of the zone.

You can increase the response time for all
movement (making you turn, swim and walk
faster):
At offset 166D48, write in either:
8FC2 F53C (for a 50% speed increase)
or 0AD7 233D (for a 100% speed increase)

Much more than 100% becomes very
disorienting when you turn, but makes moving
long distances a snap.

Lastly, you can make it start the zone at
any arbitrary location, but this is a bit
trickier than just pasting in the patches
mentioned above.

To enable this trick, patch all the
following offsets to hex 90:
A8AD-A8B2, A902-A907, A954-A959, A9A6-A9AB, A9F8-A9F9

Now you have to get the X,Y,Z coords into
the number format used in the tutorial
program. This is the IEEE-754 hex
notation with a reversed byte order.

There is a web page with a tool that turns
decimal numbers into IEEE-754 notation:

http://babbage.cs.qc.edu/courses/cs341/IEEE-754.html

It uses javascript, you type in the number
in decimal, click "ROUNDED" and it tells
you the hex code you need to write. Then
you have to REVERSE the order of what it
says. Example:

You go to the Hole, and you see the message
on the wall: Go to 800, 200, 0.

Go to the web page and type in 800, it
comes back with "44480000" (under Hexadecimal
under "Single Precision"). Now reverse the
bytes: 00 00 48 44 (don't just reverse the
individual digits). Now look up 200: it
comes back with "43480000", that becomes
00 00 48 43. Zero is still "00000000".

Locations for the coords:
X: Offsets C1E32 - C1E35
Y: Offsets C1E3C - C1E3F
Z: Offsets C1E46 - C1E49

So write 0000 4844 to C1E32, 0000 4843 to
C1E3C, and 0000 0000 to C1E46.

It's not *THAT* hard to do, really... it
just sounds difficult until you try it.

Once you've patched your tutorial.exe, you
can use EZE to explore different zones
rather than manually editing the zone each
time. Get EZE from:

http://home.tampabay.rr.com/estembler/tristram/eze.htm

Have fun...
- The Saint!

Well, one of my friends hopes to write a small windows program that will let you patch the offsets to these and other values (like, to become any race, or adjust the response time to any factor). With luck, he'll finish this over the weekend. Not sure where we'll host it though. For now, manual patching for people a bit rusty:

get AXE (Advanced Hex Editor) from:
ftp://ftp.simtel.net/pub/simtelnet/win95/editor/axe20.zip

Copy Tutorial.EXE to some other backup name, then open it up in AXE. Now you see your window divided into 3 columns; Offset, Bytes and Text. Scroll the file to the closest offset to the one you want to patch, then move your mouse over the bytes (and watch the Offset display on the status line) to get the exact byte to start modifying. Just put your cursor on that line and type in the patched
values. If it says to NOP or 90 out a range of offsets, just start at the beginning of the range and change all the hex values to 90 until (and including) the end of the range.
