Decaying Skeleton 		posted 08-07-2001 10:49 PM         
well, here's my thinking on it..

the obj file holds a lot more than just doors, right? other inanimate objects in it?

doors are, essentially, switches.. you click on them once, they open.. click again and they close.. this has to be controlled SERVER side (obviously, otherwise one person will open a door but everyone else on the server will see it as still closed and will have to open it themselves)

so the stuff that doesn't have to be controlled server side in the obj file gets loaded, and the rest doesn't, including doors..

i think doors definitely have to be specified on the server i.e their size, orientation, which way they move, speed, whether they need a key, etc.. stuff in the object file is, most likely, just the door models themselves.. so until doors get programmed into the emu server, and included in zone config files (along with spawns) i think you'll need a lot more than just poking about in the GM client to spawn working doors (or even non-working doors)

of course, i might be totally wrong but that seems to be the logical explanation..
