This is how I got SEQ to work using a dialup connection and without making the linux box share a modem.

Comp 1 - 192.168.0.1; Has the active dialup connection; Can run EQ but won't make a difference.
Comp 2 - 192.168.0.2; Runs EQ.
Comp 3 - 192.168.0.3; Running SEQ sniffing 192.168.0.2, works perfectly.

This works because comp 2 is receiving packets through the hub and nic, while it won't work on comp 1 because that's receiving packets through the modem.

Galen:

Comp 1 has its ip assigned as 192.168.0.1
Comp 2 obtains its ip automatically and comes out as 192.168.0.2
Comp 3 (Linux) has the name server set to 192.168.0.1 and obtains its ip automatically (DHCP).

You MUST have internet connection sharing (win98 SE) on the computer w/ the dialup connection for this to work.
