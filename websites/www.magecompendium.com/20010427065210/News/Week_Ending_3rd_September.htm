Subject : EQCSR - Official Everquest Patch Mailing List


Everquest and Everquest: The Ruins of Kunark have been undergoing a great number of changes and enhancements ever since their initial releases. As an Everquest player it is important to keep up with the changes, as many will have a direct impact on your character or characters. When a player logs in he or she will receive a patch message about the changes made to the world, if any, since the player last logged out. This, however, is not always enough to avoid unwanted situations with your Everquest characters.


In order to give users more warning of upcoming changes, Verant wishes to build a mailing list available only to current account holders.


This mailing list will feature, among many other things :


- Confirmed future patches


- Potential patches not yet confirmed


- Item patches


- Client patches


- Specific warnings about changes to mobile NPCs that could make them more
dangerous to engage


Subscribers to the list will be sent an email well in advance of any patches.

You are not required to sign up for the list, although it would be advised that you do so in order to keep yourself informed of all patches. It is possible that the list signup process could be integrated into the Everquest client program in the future.


In order to sign up to the mailing list and to confirm that you are a registered Everquest account holder, please send a reply to this email with "EQ MAILING LIST" in the subject line. The message should include nothing except your Everquest LOGIN NAME in the first line and PASSWORD in the second line. If you own several Everquest accounts you only need to reply for one account.

If your login name was Verant and your password was Everquest, your reply would look like this :

Verant
Everquest

Once the message has been received you will be sent a message to confirm your registration.

Topic :  /pet kill <target>

 

This came of the official EverQuest message boards.

Hi there, just learned a pet command that makes getting bone chips a cake walk for necros.

We all know about the command "/pet kill". We use it every day. But most, including myself until a day or two ago, I learned you can instruct your pet to kill certain creatures. Try going into a decaying skeleton rich area like Paineel newbie grounds, and tell you pet "/pet kill decaying." It will then run off and kill the nearest decaying skeleton, without you even targeting it.

I killed skeleton after skeleton with this technique, and would watch my skeleton run to places I couldn't even see in search a new target. It was great.

(One piece of advice, if you do this on a target that is very far away, your pet will not be able to find it, but if you give him no further commands, once the target is in range he will remember you wanted him to kill it and run off. Very good command, with limited uses, but I know it comes in handy when hunting in areas where you can't see targets very well. Your pet will see them, and race off to kill them.)

This is not a new command, but has been about since beta. Have used it myself a few times :). See In Game Commands/Pet Commands section.

Topic :  Patch

 

August 30th, 8:00am

Today's Patch

Today we modified the EverQuest client to implement the following changes :

- Corrected a bug that would cause players to crash if they entered the game, exited to the server selection screen, and tried to reenter the game.


- Corrected a bug that caused delays when switching between UI modes due to the text wrapping change implemented last week.


- The /book command (/book #) now accepts pages 1 through 50, and will switch to the proper page.


- Opening your spell book will automatically cause you to begin meditating providing that you have the skill.


- Removed the /meditate command since it is no longer necessary.


- You can now activate a hotkey while the spell book is open by pressing the corresponding number-key.


- The "/target [name]" command can be used to target other players by name. Please note that this ability is subject to the range to the target.


- EverQuest will now write the original default colors when creating the EQCLIENT.INI file, rather than the new colors that were put in a few weeks ago.


- The @-character can be used in emotes to indicate a target. For instance, if you target "a gnoll" and type "/emote laughs cruelly at @", the output will be, "Playername laughs cruelly at a gnoll".


- People with the /LFG (Looking for Group) flag set will now have their over-head name prefixed with an asterisk. Remember that you can search for people looking for a group by using the "/who" and "/who all" commands.


Typing "/who all ranger 40 50 lfg" will return a list of all rangers between 40 and 50 that are looking for a group.


- The "Sense" series (Undead, Animal, and Summoned) will no longer point you towards people's pets.

Note on Voice Recognition

Last week we implemented voice recognition technology in EverQuest. To use this new feature, please refer to the EQROKManualSupplement.doc file in your EverQuest directory. Please note that you must already have a voice recognition engine that is compliant with the Microsoft Speech API (SAPI), version 4.0. The voice recognition technology was tested with Dragon NaturallySpeaking(r) v4.0. Check with the developer of your software to find out if it is SAPI 4.0 compliant.

- The EverQuest Team

Topic :  Clipping Plane

 

This came of the official EverQuest message boards.

 

Here's the deal :

The bug with the rain is very low priority.

Slightly higher priority is having a game designer go through ALL outdoor zones and revisit the max clipping distance associated with the zone.

IF the zone doesn't have an artificially limited clip for effect (eg. Toxxulia Forest), the game designer will make the max clip the farthest it can be based on the max distance used when exporting the zone (it can't go farther, else things will cease to draw correctly).

As stated before, several months ago when I posted on the subject, this is low priority (getting FA style quests in the game, revamping lesser used dungeons, working on Velious, etc. are all higher priority).

Other than the weird rain bug and the fact that apparently several zones could indeed have their max clip increased, I do NOT believe there is any 'bug'.

Darkmatter, I know this is your personal crusade, and I respect that :)

We DID, in my opinion, get to the bottom of it: for some reason, either in code or in data, many of the zones' max clipping planes were apparently reduced at some point.

I think your problem is with the priority we are assigning this issue, and I apologize that it's not high enough for some of you.

The plan is to have a game designer go into the data for each zone and move the clipping plane maximum out as appropriate.

If that fails, we'll look at it in code (which would involve a programmer). But my inclination is that the data change will solve the problem.

Again, it's not very high on the priority list. The FA style quests are (and are near being ready for testing, btw.), as are the dungeon revamps (which will be focused on after the FA style quests).

 

- Brad McQuaid

Topic :  Link Death

 

This came of the official EverQuest message boards.

 

I appreciate the time that you took to write this up. Let me see if I can explain the current technology is as much detail.

Pets can only exist in the world when their owner also exists in the world, meaning that once the owner disappears, so does the pet. At the same time, the character goes linkdead, that character cannot reenter the game until they are removed from the world.

Once a character is removed from the world, the world has no knowledge of the existence of the character. As such, the world no longer processes data relative to the character, since the character does not exist. That is why NORENT objects are removed as the character is being removed, since the world has no knowledge of the character once they are gone.

Certain parts of the description above are core system mechanics for EverQuest that it wouldn't make sense to change after several years of getting them operating just right. That said, we do have some plans to improve the current situation in the works. I can't comment specifically on them until we have some new things implemented and tested on the Test Server.

 

- Gordon
