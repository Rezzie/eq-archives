I think what Knek at EQ-traders checked with a GM and got confirmed was not approved was modifying the HTML-based help file which is read by EQ and displayed in the help window in the new UI.

He was designing a log parser program which changed the contents of the help file to reflect up-to-date parsed data (with the parser running in the background while EQ was running). He would then open the help window and see that fresh data.

He had tested the concept by rewriting the help file HTML to display some recipes. However, modifying the help file before the game is started requires you to circumvent the patcher in order to run the game with your modified help file.

Knek petitioned to check (the smart thing to do) and got the response doing stuff like that was not allowed. He was not specific about what made it not allowed, but I would think it is more likely the problem was with how he built his solution than that it was used to show recipes.

What is shown here is not a change in a Verant patched file, and not something updated dynamically while EQ is running, just a static texture applied to one window of a custom skin. I have not (yet) heard of anyone getting in trouble for that.
