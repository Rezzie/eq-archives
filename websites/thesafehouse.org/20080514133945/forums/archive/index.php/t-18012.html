i've noticed a couple bugs with yalp. the first can cause incorrect numbers.

if you parse a fight in real time using yalp using the monitor feature on a fight with adds yalp will total only the boss numbers, and ignore the adds. however if you then load that same log from a file to parse after the fact it will include the damage from the adds into the boss fight parse. eq companion does this every fight regardless. that's why i always parse yalp in real time and post the results from there. as doing it after the fact will not give the damage totals i'm looking for.

also with yalp, odd little bug. if you parse in real time the fight time starts when you, the parser, attacks, however it still records and calculates the damage from others before you start attacking. you'd think this would skew the results as they'd have higher damage than actual fight time. but it doesnt. reloading the parse after the fight shows the correct times but still has the same dps and total damage numbers. the only number that changes is the parser's adjusted dps. but seeing as most dont post that number it hardly matters.

EQ companion starts when the first person hits the mob (I think).

EQC will show different fight durations for each person, but will start logging the fight under the mob name after the first person hits it.

The DPS calculation for each person is based on their active combat duration, not the overall event time.

So yes, it starts a timer when the first person hits, but it will use each person's timer to calculate their DPS.
